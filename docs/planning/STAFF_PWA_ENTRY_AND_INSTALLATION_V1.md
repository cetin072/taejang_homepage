# 태장 임직원 업무앱 진입·설치 기준 v1

상태: **확정**

## 1. 목적

태장 공개 홈페이지는 일반 방문객에게 회사 정보를 제공하고, 임직원은 별도의 업무앱 진입화면을 통해 로그인한 뒤 역할에 맞는 내부 화면을 사용한다. 모바일에서는 업무앱을 PWA 방식으로 홈 화면에 설치하여 앱처럼 쉽게 실행할 수 있도록 한다.

## 2. 전체 구조

- 공개 홈페이지 `/`
  - 일반 방문객 대상
  - 회사소개, 사업소개, 태장소식, 채용, 문의 등 제공
  - 로그인하지 않아도 정상 이용
- 임직원 시작화면 `/staff/` 또는 이에 준하는 경로
  - 로그인
  - 회원가입 신청
  - 승인 대기 안내
  - 업무앱 설치 안내
- 직원 업무화면 `/app/`
  - 일반 근로자, 반장, 팀장, 근로자지원, 홍보, 운영총괄, 대표이사 등 역할별 화면
- 시스템 관리화면 `/admin/`
  - 시스템 최고관리자의 가입 승인, 권한, 계정상태, 감사기록 관리

공개 홈페이지와 내부 업무플랫폼은 같은 도메인 계열을 사용할 수 있으나 인증·데이터·장애 경계는 분리한다.

## 3. 공개 홈페이지의 임직원 진입 위치

### 데스크톱

- 상단 오른쪽에 작은 `임직원` 메뉴를 둔다.
- 일반 방문객용 주요 메뉴보다 시각적 우선순위를 낮춘다.
- `회원가입`과 `로그인`을 공개 홈페이지 상단에 각각 직접 노출하지 않는다.

### 모바일

- 햄버거 메뉴 안에 `임직원 업무앱` 항목을 둔다.
- 모바일 첫 화면에 큰 로그인 버튼을 상시 노출하지 않는다.

`임직원` 메뉴를 선택하면 공개 홈페이지가 관리자 화면으로 변하는 것이 아니라 임직원 업무앱 시작화면으로 이동한다.

## 4. 임직원 업무앱 시작화면

시작화면은 다음 행동을 명확하게 제공한다.

1. 로그인
2. 처음 사용하는 직원의 회원가입 신청
3. 휴대전화 홈 화면에 업무앱 설치
4. 설치 또는 가입에 도움이 필요한 경우 담당자 안내

권장 문구:

> 태장 업무앱
>
> 오늘의 업무와 작업방법을 쉽게 확인할 수 있습니다.

주요 버튼:

- `로그인`
- `처음 사용하는 직원`
- `태장 업무앱 설치하기`

버튼은 한 화면에 과도하게 경쟁시키지 않고, 상황에 따라 우선순위를 조정한다.

## 5. 회원가입과 승인 흐름

1. 직원이 업무앱 시작화면 또는 직원용 QR코드로 접속한다.
2. `처음 사용하는 직원`을 선택한다.
3. 최소한의 가입정보를 입력한다.
4. 계정은 `pending` 상태로 생성된다.
5. 승인 대기 화면을 표시한다.
6. 시스템 최고관리자가 신원·소속을 확인한다.
7. 부서·직책·역할·근무반·지원화면 방식을 배정한다.
8. 계정을 `active`로 전환한다.
9. 로그인 시 역할에 맞는 화면으로 자동 이동한다.

사용자가 일반 근로자 화면과 관리자 화면 중 하나를 직접 선택하지 않는다.

## 6. 직원용 QR코드

직원용 QR코드는 공개 홈페이지 `/`가 아니라 임직원 업무앱 시작화면으로 직접 연결한다.

용도:

- 신규 직원 가입 안내
- 현장 설치 지원
- 공용 안내판
- 교육자료
- 직원 매뉴얼

QR코드 주변에는 다음 문구를 함께 제공한다.

> 태장 업무앱 설치 및 가입
>
> 카메라로 QR코드를 찍고 안내에 따라 진행하세요.
> 도움이 필요하면 담당자와 함께 진행하세요.

## 7. PWA 설치 기준

태장 업무앱은 초기에는 네이티브 앱이 아니라 PWA로 제공한다.

필수 구성:

- 앱 이름과 짧은 이름
- 태장 업무앱 아이콘
- 시작 주소
- 독립 실행 표시 방식
- 웹 앱 매니페스트
- 기본 서비스 워커
- 설치 가능 상태 확인
- 안드로이드 설치 안내
- 아이폰 홈 화면 추가 안내
- 로그인 상태와 보안 세션 처리

설치된 앱의 시작 주소는 공개 홈페이지가 아니라 업무앱 시작화면 또는 로그인 상태에 따른 업무앱 화면으로 한다.

## 8. 설치 후 사용자 경험

### 로그인되지 않은 경우

업무앱 아이콘 실행 → 임직원 로그인 화면

### 승인 대기 계정

업무앱 아이콘 실행 → 승인 대기 안내

### 승인된 일반 근로자

업무앱 아이콘 실행 → 오늘 화면

주요 메뉴:

- 오늘의 업무
- 작업방법
- 내 일정
- 중요공지
- 자주 보는 안내

### 반장·팀장·관리자

업무앱 아이콘 실행 → 본인 역할에 맞는 대시보드

## 9. 장애인 친화 사용 원칙

- 큰 글자와 큰 버튼
- 쉬운 문장
- 한 문장에 하나의 행동
- 첫 화면 선택지 최소화
- 역할 자동분기
- 설치·가입 초기에는 담당자 동행 권장
- 오류 발생 시 기술용어 대신 쉬운 안내
- 회원가입 완료와 승인 대기를 명확히 구분
- 공용 태블릿에서는 개인 민감정보를 표시하지 않음

## 10. 초기 MVP 제외

- 앱스토어와 플레이스토어 등록
- 네이티브 안드로이드·아이폰 앱 별도 개발
- 복잡한 푸시알림
- 완전한 오프라인 업무처리
- 백그라운드 동기화
- 위치추적
- 생산성 감시
- 상시 행동로그

필요성이 검증되면 후속 단계에서 검토한다.

## 11. 검수 기준

- 일반 방문자는 로그인 없이 공개 홈페이지를 정상 이용할 수 있다.
- 공개 홈페이지에서 임직원 업무앱 진입 위치를 쉽게 찾을 수 있다.
- 회원가입은 임직원 시작화면 안에서만 안내된다.
- 직원용 QR코드는 업무앱 시작화면으로 연결된다.
- 설치된 PWA 아이콘은 업무앱으로 바로 진입한다.
- 로그인 후 역할별 화면으로 자동 이동한다.
- 승인 대기 사용자는 내부자료를 볼 수 없다.
- 일반 근로자는 복잡한 화면 선택 없이 오늘의 업무 화면에 도달한다.
- 내부 시스템 장애가 공개 홈페이지 전체 장애로 이어지지 않는다.

## 12. 개발 순서

1. Phase 1A 인증·계정상태·역할·RLS 보안 기반 검증
2. 임직원 업무앱 시작화면과 로그인·가입 흐름
3. 역할별 자동 라우팅
4. 일반 근로자 정보게시판
5. PWA 매니페스트·아이콘·설치 안내
6. 직원용 QR코드와 현장 설치 안내자료

PWA 외형을 먼저 만들기보다 계정상태와 권한 차단을 먼저 검증한다.
