# Google Workspace 연동 원칙 v1

상태: **확정**

## 1. 목표

태장 업무 플랫폼 안에서 Google Workspace의 파일·문서·일정 기능을 자연스럽게 사용하되, 일반 근로자는 Google Workspace에 직접 접속하거나 사용법을 익힐 필요가 없도록 합니다.

## 2. 사용자 구분

### 관리자·팀장·사무직

- 플랫폼 완성 전에는 Google Workspace를 별도로 사용합니다.
- 회사 메일, 공유 드라이브, 공동 문서, 일정과 회의 기능을 사용합니다.
- 플랫폼 완성 후에는 태장 플랫폼에서 필요한 Google 파일과 일정을 연결·조회·생성할 수 있도록 단계적으로 통합합니다.

### 일반 근로자

- Google Workspace 계정을 기본적으로 발급하지 않습니다.
- Google Drive와 Calendar에 직접 접속하지 않습니다.
- 태장 플랫폼 모바일 화면에서 본인 업무, 공지, 일정과 필요한 자료만 확인합니다.
- Google에 저장된 자료가 필요할 때도 태장 플랫폼의 `작업자료 보기` 등을 통해 필요한 내용만 제공받습니다.

## 3. 시스템 역할 분리

### 태장 업무 플랫폼

- 업무지시와 배정
- 진행상태와 완료
- 공지와 도움 요청
- 부서 간 협의
- 승인·보완·결정
- 근로자지원
- 신규사업 기획과 공식 기록
- 역할별 대시보드

### Google Workspace

- 회사 이메일
- 공유 드라이브
- 문서·스프레드시트·프레젠테이션 공동작업
- 사진·영상·원본 파일 보관
- 캘린더와 회의

Google의 댓글이나 메일을 공식 승인·업무완료 기록으로 사용하지 않습니다. 공식 업무상태와 결정은 태장 플랫폼에 기록합니다.

## 4. 통합 방식

Google Workspace 화면 전체를 iframe 등으로 플랫폼에 그대로 삽입하는 방식을 기본으로 하지 않습니다. 태장 플랫폼의 디자인과 업무 흐름을 유지하면서 Google API와 파일 선택 기능을 필요한 지점에 연결합니다.

예시:

- 업무에 공유 드라이브 파일 또는 폴더 연결
- `관련 자료 보기` 버튼으로 권한이 있는 원본 열기
- 플랫폼 안에서 Google 파일 선택
- 행사·교육 일정을 Google Calendar에 등록
- 신규사업 생성 시 지정된 공유 드라이브 폴더 생성
- 완료 문서와 결과자료를 해당 업무에 연결

## 5. 단계별 도입

### 0단계: 독립 운영

- 관리자 위주로 Google Workspace 사용
- 공유 드라이브 구조와 권한 규칙 정착
- 플랫폼에는 Google 연동을 고려한 데이터 필드만 설계

### 1단계: 링크 연결

- 업무·사업·행사에 Google 파일 및 폴더 URL 연결
- 플랫폼에서 버튼으로 원본 열기
- API 없이 시작

### 2단계: 관리자 Google 로그인

- 관리자·팀장·사무직은 회사 Google 계정으로 플랫폼 로그인 가능
- Google 계정은 신원확인 수단이며 실제 직급·부서·승인권한은 태장 플랫폼에서 별도로 관리
- 일반 근로자 로그인은 별도 간편 방식 유지

### 3단계: Google Picker·Drive 연동

- 플랫폼을 벗어나지 않고 Google 파일 선택
- 업무에 파일 ID·URL·파일명 연결
- 필요한 경우 지정 폴더 업로드
- 최소 권한 범위 적용

### 4단계: Calendar 연동

- 교육·행사·회의 일정을 Google Calendar와 연결
- 일정의 날짜·시간·참석자 알림은 Google이 담당
- 업무 담당자·승인·완료상태는 태장 플랫폼이 담당

### 5단계: 선별 자동화

- 신규사업별 폴더 자동 생성
- 결과보고서와 사진 폴더 연결
- 반복적으로 필요한 파일·일정 작업만 자동화

## 6. 화면 설계 원칙

- 사용자는 태장 플랫폼을 업무 시작점으로 사용합니다.
- 플랫폼 화면에는 Google 제품명이 불필요하게 반복 노출되지 않도록 합니다.
- `Google Drive로 이동`보다 `작업자료 보기`, `관련 문서`, `행사 사진`, `최종 승인본`처럼 업무 중심 명칭을 사용합니다.
- Google 연동이 실패해도 업무지시·진행·승인 등 핵심 플랫폼 기능은 계속 동작해야 합니다.
- 파일 자체를 태장 플랫폼 DB에 중복 저장하지 않고 파일 식별정보와 업무 연결정보를 관리합니다.

## 7. 보안 원칙

- 개인 Gmail이 아니라 회사 Google Workspace 계정을 사용합니다.
- 회사 자료는 개인 내 드라이브보다 공유 드라이브를 우선합니다.
- 최소 OAuth 권한을 우선 검토합니다.
- 민감한 인사·근로자지원·계약자료는 별도 공유 드라이브와 제한 권한을 적용합니다.
- Google 접근권한과 태장 플랫폼 권한을 별도로 검증합니다.
- 퇴사 시 Google 계정·세션과 태장 플랫폼 계정·세션을 모두 차단합니다.

## 8. 개발 검수 기준

- 일반 근로자가 Google 계정 없이 핵심 업무를 수행할 수 있는가
- 관리자 사용자는 태장 플랫폼에서 필요한 파일과 일정을 자연스럽게 찾을 수 있는가
- Google과 태장 플랫폼에 업무상태를 중복 입력하지 않는가
- 공식 승인과 결정이 태장 플랫폼에 남는가
- Google 장애가 태장 플랫폼의 핵심 업무 수행을 막지 않는가
- 태장 플랫폼의 디자인과 역할별 단순성이 유지되는가
