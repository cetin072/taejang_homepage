# Google Workspace 연동 원칙 v1

상태: **확정**

## 1. 최우선 원칙

Google Workspace 연동 때문에 태장 업무 플랫폼의 핵심 개발 일정이 늦어지면 안 됩니다.

초기 버전에서는 Google API 연동을 필수 개발범위로 잡지 않습니다. 태장 플랫폼의 업무지시·진행·공지·협의·승인·근로자지원·역할별 대시보드가 먼저 안정적으로 작동해야 합니다.

Google 연동은 다음 조건을 모두 충족한 이후에만 확대합니다.

- 태장 플랫폼 핵심 기능이 실제 사용 가능한 상태일 것
- 역할별 권한과 업무 흐름이 안정화되었을 것
- 실제 운영에서 반복되는 불편이 확인되었을 것
- 연동 개발이 핵심 기능의 수정과 배포를 방해하지 않을 것

## 2. 목표

태장 업무 플랫폼 안에서 Google Workspace의 파일·문서·일정 기능을 자연스럽게 사용할 수 있는 확장 가능성은 유지하되, 일반 근로자는 Google Workspace에 직접 접속하거나 사용법을 익힐 필요가 없도록 합니다.

## 3. 사용자 구분

### 관리자·팀장·사무직

- 플랫폼 완성 전에는 Google Workspace를 별도로 사용합니다.
- 회사 메일, 공유 드라이브, 공동 문서, 일정과 회의 기능을 사용합니다.
- 초기 태장 플랫폼에서는 필요한 경우 Google 파일 또는 폴더 링크만 연결합니다.
- 더 깊은 통합은 핵심 플랫폼 안정화 이후 별도 단계로 검토합니다.

### 일반 근로자

- Google Workspace 계정을 기본적으로 발급하지 않습니다.
- Google Drive와 Calendar에 직접 접속하지 않습니다.
- 태장 플랫폼 모바일 화면에서 본인 업무, 공지, 일정과 필요한 자료만 확인합니다.
- 초기에는 Google 자료를 직접 열어야 하는 업무도 최소화합니다.

## 4. 시스템 역할 분리

### 태장 업무 플랫폼

- 현장관리자의 업무지시·배정·확인 기록
- 일반 근로자용 오늘 업무·작업방법·공지·일정 안내
- 현장 직접 대응과 필요한 지원 연결
- 부서 간 협의
- 승인·보완·결정
- 근로자지원
- 신규사업 기획과 공식 기록
- 역할별 대시보드

### Google Workspace

- 회사 이메일
- 공유 드라이브
- 문서·스프레드시트·프레젠테이션 공동작업
- 사진·영상·원본 파일 보관
- 캘린더와 회의

Google의 댓글이나 메일을 공식 승인·업무완료 기록으로 사용하지 않습니다. 공식 업무상태와 결정은 태장 플랫폼에 기록합니다.

## 5. 초기 개발범위

초기 태장 플랫폼에서 허용하는 Google 관련 기능은 다음 두 가지뿐입니다.

1. 관리자가 업무·사업·행사에 Google 파일 또는 폴더 URL을 입력
2. 권한 있는 사용자가 `관련 자료 보기` 버튼으로 원본을 새 창에서 열기

이 기능은 일반 URL 첨부 기능으로 구현하며 Google 전용 API를 요구하지 않습니다.

초기 버전에서 하지 않는 기능:

- Google 로그인 연동
- Google Picker
- Drive API 파일 검색
- 플랫폼 안에서 Google 파일 업로드
- 공유 드라이브 폴더 자동 생성
- Google 권한 자동 부여·회수
- Calendar 자동 등록·동기화
- Gmail 연동
- Google 문서 미리보기의 복잡한 내장
- Google 변경사항 자동 감지

## 6. 화면 설계 원칙

- 사용자는 태장 플랫폼을 업무 시작점으로 사용합니다.
- 링크 기능은 공통 첨부자료 기능으로 단순하게 설계합니다.
- 화면에는 `Google Drive로 이동`보다 `작업자료 보기`, `관련 문서`, `행사 사진`, `최종 승인본`처럼 업무 중심 명칭을 사용합니다.
- Google 링크가 없거나 열리지 않아도 업무지시·진행·승인 등 핵심 기능은 계속 동작해야 합니다.
- 일반 근로자의 첫 화면에 Google 메뉴를 만들지 않습니다.
- 관리자에게도 별도의 복잡한 Google 관리화면을 초기 제공하지 않습니다.

## 7. 후속 확장 순서

핵심 플랫폼 안정화 이후 실제 필요성이 확인되면 다음 순서로 한 단계씩 검토합니다.

### 후속 1단계: 관리자 Google 로그인

- 관리자·팀장·사무직의 회사 계정 로그인 편의 검토
- 태장 플랫폼의 직급·부서·승인권한은 별도 유지

### 후속 2단계: 파일 선택

- Google Picker 또는 Drive API로 파일 선택
- 업무에 파일 ID·URL·파일명 연결

### 후속 3단계: 일정 연결

- 교육·행사·회의 일정을 Google Calendar와 연결
- 업무 담당자·승인·완료상태는 태장 플랫폼이 담당

### 후속 4단계: 선별 자동화

- 신규사업별 폴더 자동 생성
- 결과보고서와 사진 폴더 연결
- 반복적으로 필요한 작업만 자동화

각 단계는 독립적으로 승인하며, 다음 단계가 자동으로 개발범위에 포함되지는 않습니다.

## 8. 보안 원칙

- 개인 Gmail이 아니라 회사 Google Workspace 계정을 사용합니다.
- 회사 자료는 개인 내 드라이브보다 공유 드라이브를 우선합니다.
- 민감한 인사·근로자지원·계약자료는 별도 공유 드라이브와 제한 권한을 적용합니다.
- Google 접근권한과 태장 플랫폼 권한을 별도로 관리합니다.
- 퇴사 시 Google 계정과 태장 플랫폼 계정을 각각 차단합니다.
- 초기 URL 연결 시에도 링크를 안다고 누구나 열 수 있는 공개 공유는 금지합니다.

## 9. 개발 우선순위 검수 기준

- Google 연동 없이도 태장 플랫폼 핵심 기능이 완전하게 동작하는가
- 일반 근로자가 Google 계정 없이 필요한 현장 안내를 확인할 수 있는가
- 초기 구현이 단순 URL 첨부 범위를 넘어가지 않는가
- Google 관련 작업이 핵심 플랫폼 일정의 선행조건이 되지 않는가
- Google과 태장 플랫폼에 업무상태를 중복 입력하지 않는가
- 공식 승인과 결정이 태장 플랫폼에 남는가
- Google 장애가 태장 플랫폼의 핵심 업무 수행을 막지 않는가
- 후속 연동이 각각 별도 선택사항으로 분리되어 있는가
