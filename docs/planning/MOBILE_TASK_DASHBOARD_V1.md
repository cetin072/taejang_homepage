# 태장 일반 근로자용 장애인 친화 업무안내 화면 v1

상태: **확정**

이 문서는 태장의 일반 근로자가 모두 장애인 근로자라는 현장 전제를 반영하여, 일반 근로자용 모바일·태블릿 화면을 업무 입력 앱이 아니라 읽기·확인 중심의 장애인 친화형 현장 안내판으로 정의합니다.

## 1. 핵심 목적

- 일반 근로자에게 업무관리 책임이나 실적입력을 요구하지 않습니다.
- 실제 작업 배정·진행·완료·문제 대응은 총괄반장, 업무반장, 현장 관리자 등 현장 책임자가 담당합니다.
- 일반 근로자 화면은 본인의 업무, 일정, 중요공지, 근무시간, 작업방법을 필요할 때 다시 확인하는 용도입니다.
- 반복 숙달을 지원하되 업무시간 중 계속 휴대전화를 보게 만들지 않습니다.
- 플랫폼의 목적은 근로자를 통제하거나 평가하는 것이 아니라 기억과 이해를 보조하는 것입니다.

## 2. 현장 운영 전제

- 업무현장에는 총괄반장, 업무반장, 관리자 1인 등이 기본적으로 상시 배치됩니다.
- 긴급하거나 힘든 상황은 앱 신고보다 현장 대응이 우선입니다.
- 일반 근로자는 현장 책임자에게 직접 도움을 요청하고, 필요한 기록은 반장·관리자가 남깁니다.
- 일반 근로자 화면은 현장 대응을 대체하지 않습니다.

## 3. 일반 근로자 화면의 성격

일반 근로자용 화면은 다음과 같은 **읽기 전용 게시판형 웹**을 기본으로 합니다.

1. 오늘의 업무
2. 작업방법
3. 내 일정
4. 중요공지
5. 자주 보는 안내

일반 근로자는 업무를 수정하거나 상태를 변경하지 않습니다.

## 4. 오늘의 업무

다음 정보만 단순하게 보여줍니다.

- 오늘 하는 작업
- 오전·오후 구분
- 작업장소
- 담당 반장
- 필요한 준비물
- 관련 작업방법 링크

직원은 시작·완료·진행률을 입력하지 않습니다.

## 5. 작업방법

작업방법은 기억이 나지 않을 때 짧게 확인하고 다시 현장 작업으로 돌아가기 위한 기능입니다.

필수 구성:

- 작업명
- 대표 사진
- 준비물
- 단계별 사진과 쉬운 문장
- 올바른 완성 모습
- 자주 하는 실수
- 핵심 주의사항
- 담당 반장

작성 기준:

- 한 문장에는 한 가지 행동만 씁니다.
- 어려운 행정용어와 전문용어를 줄입니다.
- 단계는 가능하면 3~7개로 구성합니다.
- 글보다 실제 작업 사진을 우선합니다.
- 잘못된 예시는 핵심 1~3개만 보여줍니다.

## 6. 내 일정과 근무시간

다음 내용만 표시합니다.

- 오늘 근무시간
- 이번 주 근무일
- 교육일
- 외부활동
- 휴무
- 행사일
- 근무장소 변경

일반 근로자는 일정을 직접 수정하지 않습니다.

## 7. 중요공지

전체 회사 공지를 모두 보여주지 않고, 해당 근로자가 꼭 알아야 할 내용만 선별합니다.

예:

- 작업장 변경
- 근무시간 변경
- 교육 일정
- 준비물
- 휴무 안내
- 차량 출발시간
- 안전 관련 안내

공지는 쉬운 문장과 사진 중심으로 작성합니다.

## 8. 자주 보는 안내

반복 숙달과 일상 적응을 위해 다음과 같은 내용을 제공합니다.

- 작업복 착용방법
- 출근 후 이동 장소
- 개인 물품 보관 장소
- 휴게실 위치와 이용방법
- 작업도구 위치와 반납방법
- 제품별 포장방법
- 작업대 정리방법
- 퇴근 전 확인사항

관리자는 근로자별로 자주 보는 안내를 지정할 수 있습니다.

## 9. 휴대전화 사용 원칙

이 화면은 업무시간 내내 사용하는 앱이 아닙니다.

권장 사용 시점:

- 출근 전 오늘 일정 확인
- 작업 시작 전 오늘 업무 확인
- 작업방법이 기억나지 않을 때 잠깐 확인
- 반장이나 관리자가 다시 보라고 안내했을 때
- 교육·행사·근무변경 공지 확인
- 퇴근 후 다음 근무일 확인

작업 중에는 필요할 때만 확인하고 바로 화면을 종료합니다.

## 10. 개인기기와 공용기기

- 개인 휴대전화 웹과 공용 태블릿 웹은 같은 기본 화면을 사용할 수 있습니다.
- 작업장에는 공용 태블릿 또는 안내모니터를 둘 수 있습니다.
- 공용기기에는 오늘의 작업, 작업순서, 완성 예시, 중요공지, 근무시간을 표시할 수 있습니다.
- 개인 휴대전화 사용을 강제하지 않습니다.

## 11. 장애인 친화 설계 기준

- 큰 글자와 큰 버튼
- 충분한 화면 대비
- 한 화면에 적은 선택지
- 사진·그림 중심 안내
- 쉬운 문장
- 복잡한 검색과 입력 최소화
- 뒤로가기와 화면 이동 단순화
- 일반 근로자에게 수정·삭제·관리 기능 미노출

## 12. 장애별 후속 특화 기능

### 청각장애

- 자주 쓰는 문장을 터치하면 화면 전체에 크게 표시
- 진동과 화면 중심 알림
- 문자·그림 중심 공지와 작업지시

예시 문장:

- 설명을 다시 보여주세요
- 재료가 부족합니다
- 작업이 끝났습니다
- 몸이 불편합니다
- 반장님을 불러주세요
- 천천히 말씀해 주세요
- 글로 적어 주세요

### 저시력·시각장애

- 큰 글자·고대비
- 화면 읽기 호환
- 작업내용 음성 읽기
- 확대 모드

### 지적·발달장애

- 한 화면에 한 단계
- 작업순서 사진
- 선택지 최소화
- 반복 안내
- 완료 예시 사진

### 지체장애

- 큰 터치영역
- 드래그 최소화
- 한 손 조작
- 반복 터치 감소

### 언어·의사소통 장애

- 그림·문장 선택판
- 필요사항 선택
- 선택 문장 크게 표시
- 필요 시 문장 읽어주기

장애명을 기준으로 기능을 강제로 정하지 않고, 직원별 실제 지원 필요에 따라 접근성 기능을 선택합니다.

## 13. 반장·관리자 역할

일반 근로자가 입력하지 않는 다음 업무는 반장·관리자가 처리합니다.

- 작업 배정
- 작업 시작 확인
- 작업 진행 확인
- 작업 완료 확인
- 수량·실적 기록
- 문제 발생 기록
- 업무 재배정
- 지원 필요사항 기록
- 작업방법 작성·수정
- 중요공지 등록

## 14. 작업방법 자료 관리

작업방법 자료의 기본 흐름:

1. 업무반장 또는 팀장이 실제 작업 기준으로 작성
2. 현장에서 내용과 사진 확인
3. 안전·품질 중요자료는 총괄 확인
4. 일반 근로자 화면에 게시
5. 변경 발생 시 수정일과 변경사유 기록
6. 이전 자료는 삭제하지 않고 사용 중지로 보관

자료 상태:

- 현재 사용 중
- 수정 중
- 사용 중지

## 15. 초기 버전에서 제외

- 일반 근로자의 작업 시작 터치
- 일반 근로자의 작업 완료 터치
- 작업 수량·실적 입력
- 출퇴근 입력
- 진행률 입력
- 업무 상태 변경
- 상시 알림 확인 요구
- 일반 근로자용 자유 댓글·채팅
- 근로자별 숙련도 점수·시험
- AI 자동평가
- 긴급상황 앱 신고 중심 구조

## 16. 초기 구현 범위

- 오늘의 업무
- 근무시간
- 가까운 일정
- 중요공지
- 자주 보는 안내
- 사진 중심 작업방법
- 단계별 작업순서
- 완성 예시
- 핵심 실수·주의사항
- 큰 글자·큰 버튼·쉬운 문장
- 개인 휴대전화·공용 태블릿 대응
- 일반 근로자 읽기 전용
- 관리자에 의한 직원별 자료 지정

## 17. 검수 기준

- 일반 근로자가 로그인 후 본인의 오늘 업무를 즉시 확인할 수 있는가
- 작업방법을 사진과 쉬운 문장으로 다시 확인할 수 있는가
- 시작·완료·실적입력을 일반 근로자에게 요구하지 않는가
- 화면 사용이 실제 현장작업을 방해하지 않는가
- 중요공지와 일정이 본인에게 필요한 범위로 제한되는가
- 작업방법 자료가 최신 상태인지 관리자가 확인할 수 있는가
- 일반 근로자에게 불필요한 관리정보와 민감정보가 노출되지 않는가
- 개인 휴대전화뿐 아니라 공용 태블릿에서도 사용할 수 있는가

## 18. 최종 원칙

**일반 장애인 근로자용 화면은 업무관리 앱이 아니라, 기억과 반복 숙달을 돕는 현장 안내판이다.**

실제 작업과 문제 대응은 현장에서 이루어지고, 플랫폼은 필요한 정보를 쉽고 정확하게 다시 확인할 수 있도록 지원합니다.
