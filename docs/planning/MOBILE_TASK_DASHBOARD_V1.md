# 태장 모바일 업무 대시보드 v1

상태: **확정**

이 문서는 장애인 근로자를 포함한 일반 근로자와 부서 팀장이 시스템을 공부하는 데 시간을 쓰지 않고, 각자 맡은 일을 쉽게 확인·배정·완료하도록 하는 모바일 업무 대시보드 기준을 정의합니다.

## 1. 최우선 목적

- 플랫폼의 목적은 직원에게 시스템 사용법을 익히게 하는 것이 아니라 맡은 일을 잘하게 돕는 것입니다.
- 일반 근로자는 로그인 후 본인 부서의 업무지시와 공지사항을 가장 먼저 봅니다.
- 팀장은 소속 직원에게 업무를 빠르게 배정하고 진행상태를 한눈에 확인합니다.
- 복잡한 입력, 보고서 작성, 다단계 메뉴 탐색은 최소화합니다.
- 장애인 근로자와 비장애인 일반 근로자에게 동일한 기본 구조를 적용하되 필요한 경우 개인별 화면 복잡도를 낮출 수 있습니다.

## 2. 매일 사용하는 기본 흐름

### 일반 근로자

- 출근하면 별도 업무용 앱을 여는 것처럼 모바일에서 태장 업무 플랫폼에 접속합니다.
- 첫 화면에서 당일 본인 업무와 소속 부서 공지를 바로 확인합니다.
- 업무를 확인하고 필요 시 `시작`, `완료`, `도움 필요`만 선택합니다.
- 일반 근로자에게는 매일 복잡한 메뉴를 탐색하거나 긴 내용을 입력하도록 요구하지 않습니다.

### 팀장급

- 팀장급은 배정된 PC를 기본 관리기기로 사용합니다.
- PC에서 부서 업무를 작성·배정하고 직원별 확인·진행·완료·도움 필요 상태를 확인합니다.
- 모바일은 외부 또는 현장 확인이 필요할 때 보조적으로 사용할 수 있습니다.
- 부서 간 협의, 다른 부서 지원 요청, 운영총괄 공유·결정 요청 등은 팀장급 이상에게만 역할에 맞게 제공합니다.

## 3. 일반 근로자 모바일 첫 화면

첫 화면의 우선순위는 다음 두 영역입니다.

1. **오늘 내 업무**
2. **우리 부서 공지**

그 아래에 필요한 경우 다음을 배치합니다.

- 예정 업무
- 교육·외부 일정
- 도움 요청
- 내 요청 현황
- 내 정보

### 오늘 내 업무 카드

각 업무 카드는 다음 정보만 우선 표시합니다.

- 해야 할 일
- 작업 장소
- 시작 또는 완료 예정시간
- 간단한 작업방법
- 준비물 또는 주의사항
- 담당 팀장 또는 도움을 요청할 사람

일반 근로자가 사용할 기본 동작은 다음으로 제한합니다.

- `확인했어요`
- `시작`
- `완료`
- `도움 필요`

긴 업무보고나 자유서술 입력을 기본으로 요구하지 않습니다.

## 4. 장애인 근로자 사용성 기준

- 큰 글씨와 큰 버튼을 사용합니다.
- 한 화면에 선택지를 많이 보여주지 않습니다.
- 어려운 행정용어와 상태명을 피합니다.
- 핵심 동작은 한두 번의 터치로 끝나야 합니다.
- 글만으로 이해하기 어려운 작업은 사진, 예시 이미지 또는 짧은 안내를 함께 사용할 수 있습니다.
- 직원이 시스템에 맞추는 것이 아니라 직원의 이해도와 업무 방식에 맞춰 화면을 단순화합니다.
- 본인 업무와 관계없는 부서정보·전체 재고·거래처·민감정보는 표시하지 않습니다.

## 5. 부서 팀장 화면

팀장 화면의 핵심은 다음 두 가지입니다.

1. **쉽게 업무배정하기**
2. **진행사항 한눈에 보기**

### 빠른 업무배정

팀장은 다음 순서로 업무를 배정합니다.

1. 업무 선택 또는 간단히 작성
2. 담당 직원 선택
3. 날짜·시간 또는 오전반·오후반 선택
4. 필요 시 사진·주의사항 추가
5. 배정

반복 업무는 매번 새로 쓰지 않고 기존 업무를 복사하거나 템플릿으로 재사용할 수 있어야 합니다.

### 진행현황

팀장 첫 화면에는 소속 직원별 현재 상태를 간단히 표시합니다.

- 확인 전
- 확인함
- 진행 중
- 완료
- 도움 필요
- 지연

팀장은 전체 상세기록을 열지 않아도 다음을 바로 파악할 수 있어야 합니다.

- 누가 아직 업무를 확인하지 않았는가
- 누가 진행 중인가
- 누가 완료했는가
- 누가 도움을 요청했는가
- 어떤 업무가 지연되고 있는가

## 6. 부서 간 협의와 상위 보고

부서 간 협의 기능은 일반 근로자에게 노출하지 않고 팀장급 이상에게 제공합니다.

팀장급이 사용할 기본 기능:

- 다른 부서에 지원 요청
- 요청할 업무와 필요한 날짜·시간 지정
- 관련 직원 또는 물품·자료 지정
- 요청 수락·조정·완료 상태 확인
- 운영총괄에게 `참고 공유` 또는 `결정 요청`으로 구분해 전달
- 협의 내용과 최종 결정 기록 보존

일반적인 현장 협조는 팀장 간에 바로 처리하고, 다음 경우만 운영총괄에게 올립니다.

- 부서 간 의견이 조정되지 않는 경우
- 인력·예산·일정 변경이 필요한 경우
- 납기·안전·고용유지 등 중요 위험이 있는 경우
- 대표이사 판단이 필요한 사안

대시보드에는 모든 협의 내역을 나열하지 않고, 본인이 처리해야 할 미응답·지연·결정 필요 협의만 표시합니다.

## 7. 부서 공지

- 일반 공지는 부서원 전체에게 한 번에 보냅니다.
- 중요한 공지는 확인 여부를 표시합니다.
- 공지는 짧고 쉬운 문장을 우선합니다.
- 긴 문서가 필요한 경우 요약을 먼저 보여주고 상세내용은 선택해서 열게 합니다.
- 오전반·오후반 또는 특정 직원만 선택해 공지할 수 있어야 합니다.

## 8. 알림 원칙

- 새 업무 배정
- 중요 공지
- 일정 변경
- 교육·외부행사 예정
- 팀장의 수정 요청

위 항목만 기본 알림으로 사용합니다.

불필요한 알림을 남발하지 않으며, 완료된 업무를 반복해서 재촉하지 않습니다.

## 9. 역할별 대시보드 노출 원칙

대시보드는 사용자의 모든 권한과 자료를 한 화면에 보여주는 메뉴판이 아닙니다. 각 직급이 오늘 확인하고 처리해야 할 중요한 항목만 보여줍니다.

### 일반 근로자

- 오늘 내 업무
- 소속 부서 공지
- 교육·외부 일정
- 도움 요청

### 부서 팀장

- 오늘 배정할 업무
- 미확인·진행·완료 현황
- 도움 필요·지연 업무
- 부서 간 협의 요청
- 팀장이 처리해야 할 공지와 보고

### 운영총괄

- 부서별 지연·위험
- 결정이 필요한 부서 간 협의
- 중요 승인과 인사·운영 변경
- 대표이사 보고 필요사항

### 대표이사

- 최종 승인이 필요한 안건
- 중대한 고용·안전·계약·예산 위험
- 핵심 경영정보와 예외사항

직급이 높을수록 접근 가능한 범위는 넓어지지만, 첫 대시보드에 보이는 정보는 더 압축합니다. 상세 자료는 별도 메뉴와 검색을 통해 확인합니다.

## 10. 팀장과 일반 근로자의 역할 구분

### 일반 근로자

- 본인 업무 확인
- 시작·완료 표시
- 공지 확인
- 도움 요청
- 본인 일정 확인

### 팀장

- 부서 업무 생성·배정
- 직원별 진행상태 확인
- 도움 요청 대응
- 미완료 업무 재배정
- 다음 반 또는 다음 날로 인계
- 부서 공지 작성
- 부서 간 협의와 상위 보고

일반 근로자에게 팀장용 관리 기능을 노출하지 않습니다.

## 11. 초기 개발에서 제외할 복잡성

- 일반 근로자에게 긴 일일보고 작성 요구
- 복잡한 성과평가 점수
- 다단계 결재선
- 업무마다 많은 필수 입력항목
- 일반 근로자의 전체 부서현황 열람
- 모든 기능을 한 화면에 노출
- 시스템 사용 자체를 위한 별도 교육이 많이 필요한 구조

## 12. 개발 검수 기준

- 일반 근로자가 출근 후 모바일에서 앱처럼 바로 접속할 수 있는가
- 로그인 후 본인 업무와 부서 공지를 즉시 볼 수 있는가
- 주요 동작이 한두 번의 터치로 가능한가
- 장애인 근로자도 별도 설명 없이 핵심 기능을 이해할 수 있는가
- 팀장이 PC에서 직원과 반을 선택해 빠르게 업무를 배정할 수 있는가
- 팀장이 미확인·진행·완료·도움 필요 상태를 한눈에 볼 수 있는가
- 반복 업무를 복사하거나 템플릿으로 재사용할 수 있는가
- 팀장급에게만 부서 간 협의·지원 요청·상위 보고 기능이 제공되는가
- 각 직급 대시보드가 본인이 처리할 중요한 내용만 압축해 보여주는가
- 일반 근로자에게 불필요한 관리정보와 민감정보가 노출되지 않는가
- 플랫폼이 업무를 방해하지 않고 업무수행을 돕는가
