# 태장 업무플랫폼 역할별 화면 목록·이동도 v1

상태: **확정**

## 1. 목적

이 문서는 `MVP_FUNCTIONAL_SPECIFICATION_V1.md`와 `ROLE_PERMISSION_MATRIX_V1.md`를 실제 화면 단위로 분해한다. 화면 수를 과도하게 늘리지 않고, 같은 데이터를 역할별로 다르게 보여주는 것을 기본 원칙으로 한다.

## 2. 공통 화면

### 2.1 로그인

- 이메일 또는 사번 기반 로그인
- 비밀번호 재설정
- 승인 대기 안내
- 정지·퇴사 계정 접근 차단 안내

### 2.2 회원가입

- 이름
- 연락처 또는 이메일
- 소속 확인용 최소정보
- 개인정보 수집 동의
- 가입 후 `승인 대기` 상태

가입만으로 내부자료를 열람할 수 없다.

### 2.3 내 정보

- 이름
- 부서
- 직책
- 역할
- 근무반
- 계정 상태
- 지원화면 방식
- 비밀번호 변경

일반 사용자는 부서·직책·역할·상태를 직접 변경할 수 없다.

### 2.4 접근 제한

권한이 없는 화면, 승인 전 계정, 정지·퇴사 계정에 공통 적용한다.

## 3. 일반 장애인 근로자 화면

### 3.1 홈 — 오늘

첫 화면에 다음을 표시한다.

- 이름
- 오늘 근무시간
- 오늘의 업무
- 작업장소
- 담당 반장
- 중요한 일정 또는 공지
- 작업방법 바로가기

표시 정보가 없을 때는 빈 오류화면이 아니라 `오늘 등록된 업무가 없습니다. 반장에게 확인하세요.`처럼 다음 행동을 안내한다.

### 3.2 오늘의 업무 상세

- 업무명
- 시간
- 장소
- 담당 반장
- 준비물
- 주의사항
- 작업방법 보기

시작·완료·진행률·수량·실적·출퇴근 입력은 제공하지 않는다.

### 3.3 작업방법 목록

- 오늘 하는 작업
- 자주 보는 작업
- 포장
- 검수
- 정리
- 안전
- 회사생활

초기에는 복잡한 통합검색보다 큰 분류 버튼을 우선한다.

### 3.4 작업방법 상세

- 대표 사진
- 준비물
- 3~7단계
- 한 화면 한 단계
- 올바른 완성 모습
- 핵심 주의사항
- 자주 하는 실수
- 처음부터 보기
- 오늘 업무로 돌아가기

### 3.5 내 일정

- 오늘과 가까운 근무일정
- 교육
- 외부활동
- 휴무
- 근무장소 변경
- 차량 출발시간

월간 복잡한 달력보다 가까운 일정 목록을 기본으로 한다.

### 3.6 중요공지

- 본인 대상 공지만 표시
- 제목
- 쉬운 안내문
- 준비물·장소·시간
- 사진 또는 관련 링크
- 필요한 공지만 `확인했어요`

공지 확인 여부는 평가나 실적으로 사용하지 않는다.

### 3.7 자주 보는 안내

- 출근 후 이동장소
- 쉬는 시간
- 화장실
- 개인물품 보관
- 작업복
- 도구 반납
- 몸이 불편할 때

### 3.8 일반 근로자 이동도

```text
로그인
→ 오늘
  ├─ 오늘의 업무 상세
  │   └─ 작업방법 상세
  ├─ 중요공지
  ├─ 내 일정
  └─ 자주 보는 안내
```

작업방법 확인 뒤에는 화면 사용을 계속 유도하지 않고 현장으로 돌아가는 흐름을 우선한다.

## 4. 공용 태블릿·안내화면

### 4.1 작업반 홈

- 작업반 이름
- 오늘의 업무
- 시간
- 장소
- 담당 반장
- 중요공지
- 작업방법 바로가기

### 4.2 표시 제한

공용화면에는 다음을 표시하지 않는다.

- 개인 상담·지원정보
- 개인 인사정보
- 개인 일정 중 민감한 내용
- 다른 직원의 예외 배정 사유
- 계정관리 기능

개인정보가 필요한 화면은 개인 로그인으로 전환한다.

## 5. 현장 반장·관리자 화면

### 5.1 현장 홈

우선 표시:

- 오늘 작업반
- 미배정 작업
- 다음 근무로 이어지는 작업
- 중단 작업
- 재료·장비 문제
- 안전·품질 확인
- 근로자지원 연결
- 미확인 인계

정상 완료 업무는 접어서 표시한다.

### 5.2 오늘 업무 배정

- 반복 템플릿 불러오기
- 날짜
- 오전·오후반
- 작업반
- 담당 반장
- 장소
- 당일 특이사항
- 예외 근로자 변경

작업반 전체 배정을 기본으로 하고 예외 근로자만 개별 수정한다.

### 5.3 작업 결과 기록

- 완료
- 일부 완료
- 다음 근무에 계속
- 다른 작업으로 변경
- 중단
- 필요 시 수량
- 짧은 메모

### 5.4 예외·변경 기록

- 변경 대상
- 기존 업무
- 변경 업무 또는 상태
- 변경 사유

### 5.5 문제·조치 기록

- 문제 유형
- 관련 작업
- 현장 조치
- 현재 상태
- 후속 확인 여부
- 근로자지원 연결 여부

### 5.6 인계

- 작업
- 현재 상태
- 남은 내용
- 주의사항
- 인계 대상
- 확인 여부

### 5.7 반복 업무 템플릿

- 목록
- 상세
- 새 템플릿
- 수정
- 사용 중지

과거 사용 템플릿은 삭제보다 사용 중지로 보존한다.

### 5.8 작업방법 관리

- 작업방법 목록
- 새 자료 작성
- 단계별 사진·문장 편집
- 완성 예시
- 주의사항
- 변경 사유
- 미리보기
- 게시·사용 중지

### 5.9 현장관리 이동도

```text
현장 홈
├─ 오늘 업무 배정
│   ├─ 반복 템플릿 선택
│   └─ 예외 근로자 변경
├─ 작업 결과
├─ 문제·조치
│   └─ 근로자지원 연결
├─ 인계
├─ 반복 업무 템플릿
└─ 작업방법 관리
```

## 7. 근로자지원 화면

### 7.1 근로자지원 홈

- 오늘 확인할 직원
- 신규 지원 요청
- 후속 확인 예정
- 기한 지난 조치
- 현장 협의 필요
- 고용위험 중요보고

### 7.2 근로자 목록

- 이름
- 부서·근무반
- 현재 지원상태
- 다음 확인일
- 담당자

진단명이나 장애 세부내용을 일반 목록에 노출하지 않는다.

### 7.3 근로자 지원카드

초기에는 다음 세 영역을 중심으로 한다.

1. 기본 업무정보
2. 지원·상담 기록
3. 조치·후속 확인

### 7.4 상담·지원 기록 작성

단계별 질문:

- 어떤 상황이 있었는가
- 근로자는 무엇을 말했는가
- 현장에서 무엇을 조치했는가
- 누구와 협의했는가
- 다음 조치는 무엇인가
- 담당자와 기한은 언제인가
- 언제 다시 확인하는가

### 7.5 현장 전달사항 작성

상담 원문과 분리해 업무수행에 필요한 사실·지원방법·확인기한만 전달한다.

### 7.6 후속 확인

- 예정일
- 담당자
- 실행 여부
- 현재 변화
- 추가지원 여부
- 종료 또는 계속

### 7.7 근로자지원 이동도

```text
근로자지원 홈
├─ 근로자 목록
│   └─ 근로자 지원카드
│       ├─ 상담·지원 기록
│       ├─ 조치 등록
│       ├─ 현장 전달사항
│       └─ 후속 확인
├─ 신규 지원 요청
└─ 기한 지난 조치
```

## 10. 대표이사 화면

### 10.1 대표이사 홈

- 최종 승인

- 핵심 경영정보
- 중대한 예외·위험

일상 부서업무를 기본화면에 과도하게 노출하지 않는다.

### 10.2 세부 조회

필요 시 부서별 업무·기록·첨부자료를 조회할 수 있다. 민감정보 열람에는 업무상 필요성과 감사기록을 적용한다.

## 12. 외부지도자 화면

초기에는 제한된 화면만 제공한다.

- 본인 일정
- 담당 작업·교육 안내
- 준비사항
- 관련 작업방법·자료 링크
- 관리자에게 전달할 간단한 안내

직원 인사정보, 상담기록, 전체 부서현황, 계정관리에는 접근하지 않는다.

## 14. 공통 화면 원칙

- 모바일 우선 반응형 웹
- 팀장·관리자는 PC에서도 효율적으로 사용
- 역할에 없는 메뉴는 숨기기만 하지 않고 데이터 접근도 차단
- 같은 정보를 역할에 맞게 요약해 표시
- 빈 상태·오류 상태에서 다음 행동을 안내
- 저장 후 결과와 다음 이동을 명확히 표시
- 삭제보다 사용 중지·취소·정정 우선
- 민감정보는 일반 업무 데이터와 분리
- 모든 화면에서 공개 홈페이지 장애와 내부 플랫폼 장애가 연결되지 않도록 분리

## 15. 1차 개발 검수 흐름

### 일반 근로자

```text
회원가입 → 승인 대기 → 최고관리자 승인
→ 로그인 → 오늘 업무 확인
→ 작업방법 확인 → 일정·공지 확인
```

### 최고관리자

```text
가입 승인 대기 확인
→ 사용자 신원 확인
→ 부서·직책·역할·근무반 배정
→ 승인
→ 사용자 로그인 가능 확인
```

### 관리자·팀장

```text
로그인
→ 오늘 업무·작업방법·일정·공지 작성
→ 대상 지정
→ 모바일·태블릿 미리보기
→ 게시
→ 일반 근로자 화면 반영 확인
```

## 16. 개발 완료 기준

- 권한별로 필요한 화면만 보이는가
- 권한이 없는 데이터는 직접 URL로도 열리지 않는가
- 승인 전 계정이 내부자료를 볼 수 없는가
- 김형철 최고관리자가 직책·역할·부서·근무반을 나중에 배정할 수 있는가
- 일반 근로자가 입력 없이 오늘의 업무와 작업방법을 확인할 수 있는가
- 공용화면에 개인 민감정보가 노출되지 않는가
- 관리자 작성내용이 개인 휴대전화와 공용 태블릿에 이해하기 쉽게 표시되는가
- 2차·3차 기능을 추가해도 메뉴·권한 구조를 다시 만들 필요가 없는가
