# 태장 업무플랫폼 MVP 최종 기능명세서 v1

상태: **확정**

## 1. 목적

태장 업무플랫폼 MVP는 장애인 근로자의 업무 이해와 반복 숙달을 돕고, 현장 책임자·지원부서·홍보부서·운영총괄이 역할에 맞게 업무를 관리하도록 지원한다. 플랫폼은 현장 대응을 대체하지 않으며, 공개 홈페이지와 내부 업무플랫폼의 장애 경계를 분리한다.

## 2. MVP 개발 단계

아래 `1차 개발`은 Phase 1A 전체가 순차적으로 도달할 범위다. 첫 개발 PR은 `PR17_FINALIZATION_AND_HANDOFF_V1.md`에 따라 인증·계정상태·역할·RLS·감사로그의 보안 기반 검증만 수행하고, 그 검증이 통과한 뒤 정보게시판과 관리화면을 나누어 구현한다.

### 1차 개발 전체 범위: 기반과 일반 근로자 정보게시판

- 로그인
- 가입 승인 대기
- 계정·부서·직책·역할 배정
- 계정 상태 관리
- 일반 근로자 정보게시판
- 오늘의 업무
- 작업방법
- 내 일정·근무시간
- 중요공지
- 자주 보는 안내
- 관리자 작성·수정 화면
- 기본 권한과 중요 변경 감사기록

### 2차 개발: 현장관리

- 반복 업무 템플릿
- 작업반 단위 당일 배정
- 예외 근로자 변경
- 작업 결과 기록
- 예외·변경 기록
- 문제·조치 기록
- 인계
- 근로자지원 연결

### 3차 개발: 지원·홍보·전문업무

- 근로자지원·상담 상세기록
- 후속조치와 재확인
- 홍보업무 계획·자료·검토·승인
- 일반 행정·부서 간 협업 요청
- 단계별 작성도우미
- 보완 요청과 승인 흐름

## 3. 계정과 가입

### 3.1 가입 흐름

1. 사용자가 회원가입한다.
2. 계정은 `승인 대기` 상태로 생성된다.
3. 시스템 최고관리자가 신원과 소속을 확인한다.
4. 부서·직책·역할·근무반·지원화면 방식을 배정한다.
5. 승인 후 필요한 화면과 데이터만 접근할 수 있다.

가입만으로 내부자료를 열람할 수 없다.

### 3.2 시스템 최고관리자

- 초기 시스템 최고관리자: **김형철**
- 대표이사 이영희의 경영 최종결정 권한과 시스템 최고관리자의 계정관리 권한은 구분한다.
- 마지막 최고관리자 계정은 정지·퇴사·삭제할 수 없도록 보호한다.

### 3.3 계정 상태

- pending: 승인 대기
- active: 사용 중
- suspended: 일시 정지
- departed: 퇴사
- deleted: 삭제 처리

퇴사 처리 시 로그인 차단, 기존 세션 무효화, 신규 작성·수정·업로드·배정 차단, 담당업무 재배정, 과거 기록과 감사로그 보존을 적용한다.

## 4. 일반 장애인 근로자 정보게시판

### 4.1 목적

일반 근로자 화면은 업무관리 앱이 아니라 읽기·확인 중심 현장 정보게시판이다.

### 4.2 핵심 메뉴

1. 오늘의 업무
2. 작업방법
3. 내 일정
4. 중요공지
5. 자주 보는 안내

### 4.3 첫 화면

- 이름
- 오늘 근무시간
- 오늘 업무
- 작업장소
- 담당 반장
- 중요한 일정 또는 공지
- 작업방법 바로가기

### 4.4 오늘의 업무

표시 항목:

- 업무명
- 시간
- 장소
- 담당 반장
- 준비물
- 주의사항
- 작업방법 링크

일반 근로자는 시작·진행·완료·수량·실적·출퇴근·자유보고를 입력하지 않는다.

### 4.5 작업방법

- 대표 사진
- 준비물
- 3~7단계 사진과 쉬운 문장
- 한 화면 한 단계 우선
- 올바른 완성 모습
- 핵심 주의사항
- 자주 하는 실수
- 담당 반장
- 사용 상태
- 최종 수정일과 변경 사유

일반 근로자는 읽기만 하며 수정·댓글·완료입력을 하지 않는다.

### 4.6 내 일정

- 근무시간
- 교육
- 외부활동
- 휴무
- 근무장소 변경
- 특별행사
- 차량 출발시간

월간 복잡한 달력보다 가까운 일정 목록을 우선한다.

### 4.7 중요공지

- 대상자에게 필요한 공지만 표시
- 근무시간·장소·교육·외부활동·준비물·복장·안전·휴무·이동 관련 공지
- 중요한 공지만 `확인했어요` 사용 가능
- 확인은 업무 실적이나 평가로 사용하지 않는다.

### 4.8 자주 보는 안내

- 출근 후 이동장소
- 쉬는 시간
- 화장실
- 개인물품 보관
- 작업복
- 도구 반납
- 몸이 불편할 때 행동

### 4.9 접근성

- 큰 글자와 큰 터치영역
- 쉬운 문장
- 한 문장 한 행동
- 사진·그림 중심
- 한 화면 정보량 최소화
- 고대비 옵션
- 사진 중심 옵션
- 반장과 함께 사용
- 관리자 대리 처리
- 개인 휴대전화와 공용 태블릿 대응

장애명으로 화면을 결정하지 않고 실제 필요한 지원방식으로 설정한다.

- 장소

- 준비물
- 작업방법 연결
- 쉬운 주의사항

### 5.2 일정 작성

- 일정명
- 날짜·시간

- 담당자
- 준비물
- 이동방법
- 변경·취소 상태

### 5.3 공지 작성

## 6. 현장관리 기능

### 6.1 반복 업무 템플릿

고정 정보:

- 업무명
- 업무구분
- 담당 부서
- 기본 장소
- 기본 작업반
- 담당 반장
- 준비물
- 작업방법
- 완료된 모습
- 주의사항
- 자주 발생하는 문제
- 기본 작업시간
- 기본 필요인원
- 완료 기준
- 인계 필요 여부
- 사용 상태
- 수정일과 변경 사유

당일 정보:

- 날짜

### 6.3 작업 결과 상태

### 6.4 문제 유형

- 품질 재확인
- 안전 확인
- 건강·컨디션
- 일정·인력 변경
- 근로자지원 확인 필요
- 기타

### 6.5 현장 기록 원칙

- 작업반 단위 기본 기록
- 예외 근로자만 별도 기록
- 직원별 분 단위 시간·생산성 순위·행동로그 제외
- 현장 안전과 직접 대응이 플랫폼보다 우선
- 민감한 건강·상담 원문은 기록하지 않음

### 7.1 현장 최소기록

대상: 생산·물류·환경정비·농장

- 상태
- 예외
- 현장 조치
- 인계
- 필요한 수량
- 선택적 사진

### 7.2 일반 업무기록

대상: 일반 행정·협업·자료취합

- 목적
- 요청내용
- 담당자
- 기한
- 진행내용
- 결과
- 자료 링크
- 후속업무

### 7.3 전문 상세기록

대상: 근로자지원·상담·홍보·기획·영업·지원사업

- 경위와 목적
- 구체적 사실
- 당사자 의견
- 판단과 협의
- 조치
- 담당자와 기한
- 결과
- 후속확인
- 변경·승인이력

기록역량이 부족한 직원에게 기록을 생략하게 하지 않는다. 단계별 질문, 선택항목, 예시, 필수항목 경고, 임시저장, 팀장 보완 요청을 제공한다.

## 8. 근로자지원

- 적응·지원·고용유지가 목적
- 상담 건수나 위험점수로 평가하지 않음
- 제한된 상담기록과 현장 전달사항 분리
- 현장에는 업무수행에 필요한 조정내용만 전달
- 조치 담당자·기한·후속확인 관리
- 근무시간·계약·직무·부서·징계·해고·급여의 최종결정 권한 없음

## 9. 홍보·기획·영업

필수 기록:

- 업무 목적
- 대상 독자·고객
- 핵심 메시지
- 필수 포함내용
- 참고자료
- 사진·영상·파일 링크
- 채널과 게시 예정일
- 수정·검토 의견
- 승인 상태
- 결과와 후속활용

홍보 직원 → 홍보팀장 검토 → 운영총괄 승인 → 중요 사안만 대표이사 승인 흐름을 적용한다.

## 10. 승인과 보고

일반 흐름:

직원 → 팀장 검토 → 운영총괄 최종 승인

운영총괄은 승인·완료, 보완 요청, 보류, 반려, 대표이사 상신 중 하나를 선택한다.

대표이사 상신 대상:

- 중요 계약
- 중대한 인사·조직 변경
- 중요 예산·지출
- 신사업 시작·중단
- 중요 외부협상
- 중대한 사고·분쟁·법적 위험

## 11. 부서 간 협업

- 요청부서는 수신부서 직원에게 직접 배정하지 않음
- 수신부서 팀장이 수락·조정·반려 후 담당자 배정
- 요청 → 수신팀장 검토 → 담당자 배정 → 진행 → 완료 → 요청자 확인
- 운영총괄은 부서 간 충돌·지연·자원문제에만 개입

필드:

- 제목
- 요청부서·요청자
- 수신부서
- 요청내용·이유
- 기한
- 우선순위
- 완료기준
- 관련 링크
- 담당자
- 상태

## 12. 파일과 Google Workspace

- 초기에는 Google Drive·문서·시트·캘린더 링크를 연결
- 플랫폼 직접 업로드와 Google API·OAuth·Picker·권한자동화는 후속
- 파일명, 관련업무, 단순유형, 공개·내부, 중요·최종 여부 기록
- 민감문서 원문은 일반 근로자 화면에 노출하지 않음

## 13. 감사기록

중요 변경만 기록한다.

- 계정 승인·정지·퇴사·삭제
- 역할·부서·직책·권한 변경
- 중요 공지·작업방법의 게시·수정·사용중지
- 승인·반려·보완·대표이사 상신
- 민감기록 열람·수정

모든 일반 화면 클릭과 행동을 기록하지 않는다.

## 14. 초기 제외 범위

- AI 업무배정·평가·상담판단
- 직원별 생산성·숙련도·위험도 점수
- GPS·생체인증·실시간 위치
- 법정 급여·근태 자동계산
- 자동 재고차감·ERP 연동
- 자동 SNS 게시·고급 홍보분석
- 복잡한 Google API·OAuth
- 업무 채팅·자유게시판
- 영상 자동분석·대용량 스트리밍
- 복잡한 다단계 승인

## 15. 공통 검수 기준

- 가입 전 내부정보가 노출되지 않는가
- 역할·부서·직책에 따라 필요한 정보만 보이는가
- 일반 근로자가 입력 없이 오늘 업무와 작업방법을 확인할 수 있는가
- 현장관리 기록이 실제 작업을 방해하지 않는가
- 지원·상담·홍보 기록에서 필수내용 누락을 막는가
- 민감정보와 현장 전달정보가 분리되는가
- 퇴사자의 접근이 즉시 차단되고 기록은 보존되는가
- 공개 홈페이지 장애가 내부 플랫폼 장애로 이어지지 않고 반대도 동일한가
- 초기 MVP 제외기능이 설명 없이 추가되지 않는가
