# 현장 관리자·반장 운영기록 기준 v1

상태: **확정**
적용 범위: 생산·물류·환경정비·농장 등 일반 근로자가 참여하는 현장 업무의 관리자 화면

## 1. 목적

일반 근로자에게 진행입력을 전가하지 않고, 현장 책임자가 실제 작업 운영에 필요한 최소 기록만 남기게 한다. 기록의 목적은 감시·점수화가 아니라 업무 누락 방지, 인계, 작업방법 유지와 필요한 지원 연결이다.

이 문서의 최소기록 원칙은 현장 업무에 한정한다. 근로자지원·상담·홍보·기획·외부사업 등 정확성과 내용 누락 방지가 중요한 업무에는 별도의 상세 기록 기준을 적용한다.

## 2. 역할

- 총괄반장: 공통 일정·안전·이동 안내와 긴급 전달
- 업무반장: 작업반 운영, 작업 순서 안내, 완료 확인과 작업방법 관리
- 현장 관리자: 당일 배정, 예외 변경, 최소 기록, 다음 반·다음 날 인계
- 팀장: 반복 템플릿 관리와 부서 운영 확인
- 근로자지원 담당자: 지원 필요사항 협의와 후속 확인

직책 명칭이 달라도 위 책임을 실제 담당자에게 배정한다. 반장·관리자는 인사결정이나 상담 원문 열람 권한을 갖지 않는다.

## 3. 초기 운영 흐름

1. 반복 업무 템플릿을 불러와 작업반 단위로 오늘 업무를 배정한다.
2. 예외가 있는 근로자만 별도로 변경한다.
3. 현장에서 시작·진행·완료와 문제를 직접 확인한다.
4. 필요한 경우에만 결과, 예외, 조치, 지원 연결, 인계사항을 간단히 기록한다.
5. 다음 반 또는 다음 날에 필요한 정보만 인계한다.

모든 행동을 시간 단위로 기록하거나 모든 근로자별 상세 실적을 입력하도록 요구하지 않는다.

## 4. 반복 업무 템플릿

반복 업무는 매일 새로 작성하지 않고 기준 템플릿을 불러온 뒤 당일 정보만 조정한다.

### 4.1 템플릿 기준정보

- 업무명
- 업무구분
- 담당 부서
- 기본 작업장소
- 기본 작업반
- 담당 반장
- 업무 한 줄 설명
- 준비물
- 작업방법 자료 연결
- 완료된 모습
- 핵심 주의사항
- 자주 발생하는 문제
- 기본 작업시간
- 기본 필요인원 또는 권장 인원 범위
- 일반적인 완료 기준
- 인계 필요 여부
- 사용 상태: 사용 중 / 수정 중 / 사용 중지
- 최종 수정일
- 변경 사유

### 4.2 당일 업무 생성 시 확인사항

- 날짜
- 오전반·오후반 또는 시간대
- 대상 작업반
- 대상 근로자
- 담당 반장
- 작업장소
- 당일 특이사항
- 작업방법 변경 여부

작업반 전체 선택을 기본으로 하고, 예외가 있는 근로자만 별도로 변경한다. 업무 템플릿에는 특정 근로자의 장애정보, 평가, 능력순위나 민감한 지원내용을 기록하지 않는다.

## 5. 현장 최소기록 유형

초기 현장기록은 다음 네 종류로 제한한다.

### 5.1 작업 결과

- 결과: 완료 / 일부 완료 / 다음 근무에 계속 / 다른 작업으로 변경 / 중단
- 필요한 경우에만 완료·남은·불량·재작업 수량과 단위
- 필요한 경우에만 짧은 메모

직원별 생산량은 기본적으로 기록하지 않는다.

### 5.2 예외·변경

- 변경 대상
- 기존 업무 또는 상태
- 변경 업무 또는 상태
- 변경 사유

변경 사유는 재료 부족, 장비 문제, 일정 변경, 인력 조정, 교육·외부활동, 건강·컨디션, 작업 이해 지원, 기타 정도로 단순화한다.

### 5.3 문제·조치

- 문제 유형
- 관련 작업
- 현장에서 한 조치
- 현재 상태
- 후속 확인 필요 여부와 시점

문제 자체보다 어떤 지원과 조치를 했는지를 중심으로 기록한다. 건강·장애·상담의 상세내용은 작성하지 않는다.

### 5.4 인계

- 인계 대상 작업
- 현재 상태
- 남은 내용
- 주의사항
- 관련 자료
- 인계받을 반 또는 담당자

### 5.5 공통 자동정보

- 작성자와 작성시간
- 최종 수정자와 수정시간
- 중요도: 일반 / 확인 필요

중요 기록은 삭제보다 취소 또는 정정 상태로 보존한다.

## 6. 작업방법 자료

초기 자료에는 다음을 포함한다.

- 작업명과 대표 사진
- 준비물
- 3~7단계 사진과 쉬운 문장
- 올바른 완성 모습
- 핵심 주의사항과 자주 하는 실수
- 담당 반장
- 현재 사용 여부
- 최종 수정일과 변경 사유

업무반장 또는 팀장이 실제 현장 기준으로 작성한다. 안전·품질에 영향을 주는 중요 자료만 총괄 확인을 받는다. 긴 PDF 원문은 관리자용으로 보관하고, 일반 근로자에게는 쉬운 핵심자료를 별도로 제공한다.

## 7. 기록 최소화와 지원 연결

현장 문제는 먼저 현장에서 직접 대응한다. 도움이 더 필요하면 근로자지원 담당자에게 다음의 업무 수행 관련 사실만 전달한다.

- 대상 근로자
- 발생한 상황
- 현장에서 한 조치
- 추가 확인이 필요한 이유
- 확인 희망 시점

상담 경위, 진단명, 장애 세부내용, 약 복용내용과 민감정보는 공유하지 않는다.

초기 MVP에서 퀴즈, 시험, 숙련도 자동평가, AI 분석, 복잡한 다단계 현장 승인, 세밀한 행동 로그, 실시간 위치, 개인별 생산성 순위는 제외한다.

## 8. 관리자 첫 화면 우선순위

모든 기록을 나열하지 않고 다음 예외사항을 우선 표시한다.

- 미배정 작업반
- 다음 근무로 이어지는 작업
- 중단된 작업
- 재료·장비 문제
- 안전·품질 확인 필요
- 근로자지원 연결
- 인계 미확인
- 총괄 확인 필요

정상 완료된 업무는 접거나 지난 기록에서 확인한다.

## 9. 검수 기준

- 반복 업무를 템플릿과 작업반 단위로 운영하는가
- 예외 근로자만 별도 처리하는가
- 일반 근로자에게 진행·실적 입력을 요구하지 않는가
- 현장 팀장·반장의 기록이 실제 작업을 방해하지 않는가
- 작업방법 자료의 현행성·변경이력과 현장 책임자가 명확한가
- 인계와 지원 연결에 필요한 최소 기록만 남기는가
- 상담·홍보·기획 등의 상세 기록을 현장 최소기록 원칙으로 축소하지 않는가
