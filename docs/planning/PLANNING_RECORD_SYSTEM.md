# 태장 업무 플랫폼 기획 기록 체계

이 문서는 ChatGPT 기획 대화에서 확정된 내용을 제작·코딩 단계까지 누락 없이 전달하기 위한 기록 규칙입니다.

## 1. 기본 원칙

- 대화만으로 확정사항을 보관하지 않습니다.
- 하나의 기획 단락이 확정되면 해당 모듈의 기획 문서에 즉시 기록합니다.
- 확정되지 않은 아이디어는 `검토 중`으로 표시하며 개발 기준으로 사용하지 않습니다.
- 사용자에게서 `확정`, `그대로 진행`, `수정 불필요`, `좋다` 등의 명시적 승인이 나온 항목만 `확정`으로 기록합니다.
- 기존 확정사항을 바꿀 때는 이전 내용을 지우기만 하지 않고 변경 이유와 날짜를 남깁니다.

## 2. 기록 상태

각 기획 항목은 아래 상태 중 하나를 가집니다.

- `검토 중`: 논의 중이며 개발에 반영하지 않음
- `확정`: 사용자 승인 완료, 개발 필수 반영
- `보류`: 필요성은 있으나 현재 모듈 범위에서 제외
- `폐기`: 채택하지 않기로 결정
- `변경`: 이전 확정사항을 새 결정으로 대체

## 3. 기획 단락이 끝날 때 기록할 내용

1. 기획 주제
2. 사용자와 역할
3. 확정된 업무 흐름
4. 화면 구성과 상태 용어
5. 권한과 승인선
6. 자동 처리와 사람이 판단할 부분
7. 예외 상황
8. 현재 개발 범위
9. 후속 단계로 미룬 항목
10. 검수 기준

## 4. 개발 시작 전 필수 절차

Codex 또는 개발자는 다음 순서로 확인합니다.

1. `PROJECT_CHARTER.md`
2. 해당 모듈의 `docs/planning/` 확정 기획 문서
3. 관련 요구사항·화면·데이터·보안 문서
4. 현재 브랜치와 PR 범위

작업 지시문에는 반드시 다음을 포함합니다.

- 적용할 확정 기획 문서 경로
- 이번 작업에 포함할 항목
- 보류하거나 변경하지 않을 항목
- 상태 용어와 승인 흐름
- 테스트·검수 항목

## 5. 개발 완료 후 대조 검사

PR 완료 전 확정 기획 문서의 각 항목을 아래 방식으로 대조합니다.

- `구현 완료`
- `부분 구현`
- `미구현·후속 작업`
- `의도적으로 제외`

확정 항목이 누락되었으면 PR을 완료 처리하지 않습니다.

## 6. 현재 운영 방식

- 최상위 사업 방향은 `PROJECT_CHARTER.md`에 기록합니다.
- 모듈별 상세 기획은 `docs/planning/`에 기록합니다.
- 기획 변경 이력은 각 문서의 `결정 이력`에 누적합니다.
- 개발 작업은 해당 기획 문서를 인용해 기능 브랜치와 Draft PR에서 진행합니다.
- 사용자 승인 전에는 `main`에 병합하지 않습니다.

## 7. 대화 중 사용할 확인 문구

하나의 단락이 확정되면 ChatGPT는 다음을 명확히 알립니다.

> 이 단락은 확정되었으며 기획 문서에 기록합니다.

기록 후에는 변경한 문서와 핵심 확정사항을 사용자에게 보고합니다.
