# 홍보직원 대시보드 v1 확정 기획

- 상태: `확정`
- 현재 개발 대상: 태장 업무 플랫폼의 첫 운영 모듈인 홍보 모듈
- 원칙: 홍보직원은 작성과 자료 입력에 집중하고, 중요도·공개 위험·승인선 판단은 시스템과 관리자가 담당합니다.

## 1. 홍보직원의 하루

출근 후 태장 업무 플랫폼에 본인 계정으로 로그인합니다.

1. 오늘의 홍보 업무 확인
2. 직접 촬영 또는 다른 팀이 카카오톡으로 전달한 자료 확인
3. 사진 선별과 자료 정리
4. 홈페이지 자체 글 작성 또는 외부 SNS 게시
5. 외부 SNS 콘텐츠를 홈페이지 링크 카드로 등록
6. 임시저장과 미리보기
7. 승인 요청
8. 승인 대기 또는 `보완 필요` 처리
9. 게시 완료 확인
10. 퇴근 전 오늘의 완료 현황 확인

플랫폼 기록이 일일 업무 결과가 되며 별도의 중복 업무일지 작성을 요구하지 않습니다.

## 2. 대시보드 배치 순서

1. 인사말과 날짜
2. 오늘 해야 할 일
3. `보완 필요`와 오늘 마감
4. 빠른 작업
5. 촬영·게시 일정
6. 승인 현황
7. 최근 작업
8. 오늘 완료 현황
9. 이번 달 기록과 대표 사진

행동이 필요한 항목을 성과 통계보다 위에 배치합니다.

## 3. 오늘 해야 할 일 생성 방식

초기 버전에서는 다음 세 가지를 지원합니다.

- 정기 업무 자동 생성
- 홍보팀장 업무 배정
- 홍보직원 직접 추가

행사 일정에서 업무를 자동 생성하는 기능은 후속 단계로 보류합니다.

## 4. 빠른 작업

- 홈페이지 글쓰기
- 외부 콘텐츠 등록
- 내 콘텐츠 관리
- 사진·자료 정리

원본 사진 전체를 플랫폼에 저장하지 않습니다.

- 원본 사진: 별도 공유 드라이브 또는 내부 공유 폴더
- 플랫폼: 대표 이미지와 게시용 선별 사진

## 5. 직원 화면 상태 용어

- `작성 중`: 아직 작성하거나 정리하는 상태
- `승인 대기`: 관리자가 확인 중이며 직원의 즉시 행동이 필요하지 않은 상태
- `보완 필요`: 관리자가 의견과 함께 되돌려 직원이 다시 수정해야 하는 상태
- `게시 예정`: 승인 완료 후 게시 시점 대기
- `게시 완료`: 공개 완료
- `숨김`: 공개를 일시 중지
- `보관`: 운영 목록에서는 제외하고 기록 유지

직원이 `승인 요청` 버튼을 누른 이후 화면 상태는 `승인 대기`로 표시합니다.

## 6. 승인 흐름

### 일반 콘텐츠

홍보직원 작성 → 홍보팀장 확인 → 게시

### 중요 콘텐츠

홍보직원 작성 → 홍보팀장 확인 → 운영총괄 최종 승인 → 게시

### 대표이사 확인 콘텐츠

홍보직원 작성 → 홍보팀장 확인 → 운영총괄 확인 → 대표이사 확인 → 게시

대표이사 확인은 대표이사 명의 글, 중대한 공식 입장, 주요 모회사 발표, 중요한 협약·계약·경영수치, 사고·분쟁·언론 대응 등에 제한합니다.

## 7. 홍보직원의 판단 범위 최소화

홍보직원에게 아래 판단을 맡기지 않습니다.

- 일반 콘텐츠와 중요 콘텐츠 구분
- 운영총괄 또는 대표이사 승인 필요 여부
- 법률적으로 위험한 표현인지 여부
- 모회사 관련 정보의 공개 가능 여부
- 민감한 개인정보나 공식 경영수치의 공개 적절성

직원은 사실과 자료를 입력하며 판단이 어려울 때 `잘 모르겠음`을 선택할 수 있습니다. `잘 모르겠음`은 오류가 아니라 관리자 확인이 필요한 정상 상태입니다.

## 8. 직원 작성 화면의 간단한 확인 항목

- 글 종류
- 작성 명의
- 관련 기관
- 사람이 나온 사진: 있음 / 없음 / 잘 모르겠음
- 숫자·금액 포함: 있음 / 없음 / 잘 모르겠음
- 게시 희망일

직원에게 긴 위험 체크리스트나 중요도 선택 화면을 제공하지 않습니다.

## 9. 시스템의 승인선 자동 제안

시스템은 콘텐츠 유형과 입력값으로 승인선을 제안합니다.

- 보도자료 → 운영총괄 승인 추가
- 대표이사 명의 → 대표이사 확인 추가
- 모회사 관련 새로운 정보 → 운영총괄 승인 추가
- 인물 사진 있음 또는 잘 모르겠음 → 팀장 공개 확인 표시
- 숫자·금액 있음 또는 잘 모르겠음 → 운영총괄 확인 표시
- 일반 행사 후기·승인된 반복 형식 → 팀장 승인

AI가 단독으로 승인선을 확정하지 않습니다. 규칙 기반 제안 후 홍보팀장이 최종 확인합니다.

## 10. 홍보팀장의 역할

- 사실관계와 표현 확인
- 사진과 공개 가능 여부 확인
- 일반 콘텐츠 승인·게시
- `보완 요청`과 구체적인 수정 의견 작성
- 중요 콘텐츠를 운영총괄에게 전달
- 시스템 분류가 불충분한 경우 승인선 상향

위험 조건이 있는 콘텐츠를 임의로 일반 콘텐츠로 낮추지 못하도록 시스템 제한을 둡니다.

## 11. 직원 대시보드 핵심 카드

### 행동 필요

- 보완 필요
- 오늘 마감
- 임시저장

### 승인 현황

- 승인 대기
- 게시 예정
- 게시 완료

### 최근 작업

콘텐츠 제목, 콘텐츠 종류, 현재 상태, 최근 수정 시각을 표시하고 바로 열기·수정이 가능해야 합니다.

### 기록과 성취

오늘, 이번 주, 이번 달, 올해, 누적 콘텐츠 수와 최근 대표 사진을 화면 하단에 표시합니다. 평가나 직원 간 순위가 아니라 기록의 성취감을 제공하는 용도입니다.

## 12. 초기 버전에서 제외

- SNS 자동 게시
- 카카오톡 자동 수집
- 조회수·좋아요·전환율 등 고급 성과 분석
- 개인 점수·직원 순위
- 원본 사진 전체 보관
- 대표이사·운영총괄의 회사 전체 기능을 직원 화면에 혼합
- 행사 일정 기반 업무 자동 생성

## 13. 개발 검수 기준

- 직원은 중요도를 직접 선택하지 않아도 승인 요청을 완료할 수 있어야 합니다.
- `잘 모르겠음`을 선택하면 관리자 확인 흐름으로 안전하게 연결되어야 합니다.
- `보완 필요`가 `승인 대기`보다 높은 우선순위로 표시되어야 합니다.
- 일반 콘텐츠와 중요 콘텐츠의 승인선이 규칙에 따라 달라져야 합니다.
- 직원은 자신이 작성하거나 배정받은 업무만 기본적으로 확인해야 합니다.
- 대시보드는 평가보다 누락 방지와 업무 수행을 지원해야 합니다.
- 공개 홈페이지와 내부 업무 데이터의 권한·보안 경계를 분리해야 합니다.
