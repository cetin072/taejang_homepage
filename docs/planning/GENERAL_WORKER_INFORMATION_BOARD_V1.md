# 일반 장애인 근로자용 정보게시판 기준 v1

상태: **확정**
기준 문서: 이 문서는 일반 근로자 화면의 목적·행동 범위·접근성에 관한 최우선 기준이다.

## 1. 목적

태장의 일반 근로자는 기본적으로 장애인 근로자다. 일반 근로자 화면은 사무직 업무관리 앱의 축소판이 아니라, 필요한 정보를 다시 확인해 기억과 반복 숙달을 돕는 **읽기·확인 중심 현장 정보게시판**이다.

근로자는 다음을 확인한다.

- 오늘의 업무
- 근무시간과 가까운 일정
- 중요공지
- 작업방법
- 준비물과 주의사항
- 자주 보는 안내

작업 중 지속적인 휴대전화 사용을 유도하지 않는다. 개인 휴대전화와 현장 공용 태블릿·안내화면에서 같은 웹 화면을 사용할 수 있게 설계한다.

## 2. 일반 근로자에게 요구하지 않는 입력

초기 MVP에서 일반 근로자에게 다음 행동을 기본으로 요구하지 않는다.

- 작업 시작·완료 터치
- 진행률·작업수량·실적 입력
- 출퇴근 입력
- 자유서술 업무보고
- 작업 중 상시 알림 확인
- 댓글·채팅·퀴즈·시험
- 숙련도 점수·자동평가·AI 분석

긴급상황, 몸이 힘든 상황, 업무 이해가 어려운 상황은 플랫폼 신고보다 현장관리자에게 직접 알리는 대응이 우선이다.

## 3. 현장과 플랫폼의 역할 분리

현장에는 총괄반장, 업무반장, 현장 관리자 1명이 기본적으로 함께하며 필요 시 근로자지원 담당자가 연결된다.

- 총괄반장: 공통 안내, 이동·휴식·일정과 긴급 전달
- 업무반장: 작업 순서 안내, 현장 확인, 작업방법 자료 작성·수정
- 현장 관리자: 작업반 운영, 예외 변경, 최소 기록과 인계
- 근로자지원 담당자: 적응·지원이 필요한 경우 현장과 협의

작업 배정, 시작·진행·완료 확인, 수량·실적, 문제 대응과 필요한 기록은 현장 책임자가 관리자 화면에서 처리한다. 일반 근로자 화면은 현장 대응을 대체하지 않는다.

## 4. 초기 메뉴

초기 핵심 메뉴는 다음 다섯 가지다.

1. 오늘의 업무
2. 작업방법
3. 내 일정
4. 중요공지
5. 자주 보는 안내

확인은 출근 전 또는 작업 전, 일정·근무시간 확인, 작업방법을 잠깐 다시 볼 때, 반장·관리자가 안내할 때에 한정한다. 자료를 본 뒤에는 현장 작업으로 돌아가는 흐름을 우선한다.

## 5. 첫 화면 구조

로그인 후 첫 화면에는 다음 순서로 표시한다.

1. 이름
2. 오늘 근무시간
3. 오늘의 업무
4. 작업장소
5. 담당 반장
6. 작업방법 보기
7. 오늘 꼭 확인해야 하는 중요공지 또는 가까운 일정

진행률, 실적, 통계와 다른 직원의 업무는 표시하지 않는다.

## 6. 오늘의 업무

업무카드에는 다음만 표시한다.

- 업무명
- 작업시간
- 작업장소
- 담당 반장
- 준비물
- 연결된 작업방법
- 특별 주의사항

업무가 여러 개이면 시간순으로 표시한다. 한 번에 긴 설명을 펼치지 않고 필요한 항목을 선택해 본다.

## 7. 작업방법 화면

작업방법 첫 화면에는 다음을 표시한다.

- 작업명
- 대표 사진
- 준비물
- 작업순서 보기
- 완성된 모습 보기
- 주의사항 보기

작업순서는 한 화면에 한 단계씩 표시하는 것을 기본으로 한다. 각 단계는 사진과 한 문장 한 행동 원칙을 적용하고, 마지막에는 올바른 완성 모습을 보여준다.

작업방법 목록은 복잡한 검색보다 큰 분류 버튼을 우선한다.

- 오늘 하는 작업
- 포장
- 검수
- 정리
- 안전
- 회사생활

자주 보는 자료는 근로자가 직접 복잡하게 관리하기보다 반장·관리자가 우선순위를 지정할 수 있다.

## 8. 내 일정

월간 달력보다 가까운 일정 목록을 우선한다.

- 오늘과 다음 근무시간
- 교육
- 외부활동
- 휴무
- 근무장소 변경
- 특별행사
- 차량 출발시간

근로자는 일정을 수정하지 않는다.

## 9. 중요공지

본인에게 필요한 공지만 표시한다.

- 근무시간 변경
- 작업장소 변경
- 교육
- 외부활동
- 준비물과 복장
- 안전
- 휴무
- 차량 이동

중요공지만 `확인했어요` 처리를 사용할 수 있다. 이 확인은 업무 시작·완료가 아니라 안내 확인 의미다. 모든 공지에 확인을 요구하거나 반복 알림으로 압박하지 않는다.

## 10. 자주 보는 안내

반복해서 확인하는 현장생활 안내를 제공한다.

- 출근 후 이동 장소
- 개인 물품 보관 장소
- 쉬는 시간
- 화장실 위치
- 작업복 착용
- 도구 반납
- 퇴근 전 정리
- 몸이 불편하거나 어려울 때 알릴 사람

플랫폼 신고보다 가까운 반장·관리자에게 직접 알리는 행동을 안내한다.

## 11. 근무시간 전·후와 외부활동

근무시간 전에는 오늘의 업무 제목, 근무시간과 일정을 볼 수 있다. 실행 기능은 두지 않는다.

근무 종료 후에는 오늘 업무를 계속 쌓아두기보다 다음 근무시간과 가까운 일정을 표시한다.

야외공원 청소 등 외부활동은 다음을 한 카드에 정리한다.

- 날짜
- 모이는 시간과 장소
- 준비물과 복장
- 이동방법
- 담당자
- 작업방법과 안전안내 연결

## 12. 접근성 원칙

- 큰 글자와 큰 터치영역
- 쉬운 문장, 한 문장 한 행동
- 한 화면에 적은 정보와 선택지
- 사진·그림 중심 작업안내
- 복잡한 입력·검색·선택 최소화
- 개인정보와 장애 세부정보의 불필요한 노출 금지

장애명으로 화면을 일괄 결정하지 않는다. 실제 필요한 지원방식에 따라 기본 쉬운 화면, 사진 중심, 큰 글자·고대비, 반장과 함께 사용, 관리자 대리 처리, 공용 태블릿 우선 등을 선택한다.

## 13. 개인기기와 공용기기

개인 휴대전화에서는 본인 업무와 개인 일정까지 표시할 수 있다.

공용 태블릿·안내화면에서는 작업반의 오늘 업무, 작업방법과 공통공지 중심으로 표시한다. 개인 일정, 개인 지원정보와 민감정보는 공용화면에 표시하지 않는다.

## 14. 관리자 작성 항목

### 오늘의 업무

- 업무명
- 시간
- 장소
- 담당 반장
- 준비물
- 작업방법
- 주의사항

### 일정

- 일정명
- 날짜와 시간
- 장소
- 대상
- 준비물
- 이동방법

### 공지

- 제목
- 쉬운 안내문
- 대상
- 표시기간
- 중요 여부
- 사진
- 확인 필요 여부

### 자주 보는 안내

- 안내 제목
- 쉬운 설명
- 사진
- 대상
- 표시 순서

관리자 작성 화면은 긴 문장 경고, 쉬운 문장 예시, 필수항목 누락 확인, 사진 미등록 안내와 휴대전화·태블릿 미리보기를 제공할 수 있다. AI 자동작성은 초기 필수가 아니다.

## 15. 후속 후보

청각장애인용 문장판·전체화면 큰 글자, 진동·화면 중심 알림, 저시력 확대·고대비, 화면 읽기 호환·음성 안내, 그림 중심 의사소통판은 후속 검토 대상이다. 초기 MVP 필수기능으로 확대하지 않는다.

## 16. 초기 제외

- 작업 시작·완료
- 실적과 수량 입력
- 출퇴근 체크
- 다른 직원 업무
- 전체 부서현황
- 결재
- 상담기록과 인사정보
- 업무 채팅과 자유게시판
- 전체 파일보관함
- 복잡한 검색과 통계
- 평가점수와 AI 기능

## 17. 검수 기준

- 일반 근로자가 입력 없이 오늘의 업무와 작업방법을 확인할 수 있는가
- 현장 확인과 관리자 기록의 역할이 분리되는가
- 화면이 현장 작업을 방해하거나 상시 휴대전화 사용을 유도하지 않는가
- 개인기기와 공용기기에서 같은 정보가 이해하기 쉽게 보이는가
- 공용기기에 개인정보와 지원정보가 노출되지 않는가
- 지원방식은 개인 필요에 맞추되 민감정보를 노출하지 않는가
- 접속 후 오늘 업무 확인, 필요 시 작업방법 확인, 화면 종료, 현장 복귀의 짧은 흐름이 가능한가
