# 일반 장애인 근로자용 정보게시판 기준 v1

상태: **확정**  
기준 문서: 이 문서는 일반 근로자 화면의 목적·행동 범위·접근성에 관한 최우선 기준이다.

## 1. 목적

태장의 일반 근로자는 기본적으로 장애인 근로자다. 일반 근로자 화면은 사무직 업무관리 앱의 축소판이 아니라, 필요한 정보를 다시 확인해 기억과 반복 숙달을 돕는 **읽기·확인 중심 현장 정보게시판**이다.

근로자는 다음을 확인한다.

- 오늘의 업무
- 근무시간과 가까운 일정
- 중요공지
- 작업방법
- 준비물과 주의사항
- 자주 보는 안내

작업 중 지속적인 휴대전화 사용을 유도하지 않는다. 개인 휴대전화와 현장 공용 태블릿·안내화면에서 같은 웹 화면을 사용할 수 있게 설계한다.

## 2. 일반 근로자에게 요구하지 않는 입력

초기 MVP에서 일반 근로자에게 다음 행동을 기본으로 요구하지 않는다.

- 작업 시작·완료 터치
- 진행률·작업수량·실적 입력
- 출퇴근 입력
- 자유서술 업무보고
- 작업 중 상시 알림 확인
- 댓글·채팅·퀴즈·시험
- 숙련도 점수·자동평가·AI 분석

긴급상황, 몸이 힘든 상황, 업무 이해가 어려운 상황은 플랫폼 신고보다 현장관리자에게 직접 알리는 대응이 우선이다.

## 3. 현장과 플랫폼의 역할 분리

현장에는 총괄반장, 업무반장, 현장 관리자 1명이 기본적으로 함께하며 필요 시 근로자지원 담당자가 연결된다.

- 총괄반장: 공통 안내, 이동·휴식·일정과 긴급 전달
- 업무반장: 작업 순서 안내, 현장 확인, 작업방법 자료 작성·수정
- 현장 관리자: 작업반 운영, 예외 변경, 최소 기록과 인계
- 근로자지원 담당자: 적응·지원이 필요한 경우 현장과 협의

작업 배정, 시작·진행·완료 확인, 수량·실적, 문제 대응과 필요한 기록은 현장 책임자가 관리자 화면에서 처리한다. 일반 근로자 화면은 현장 대응을 대체하지 않는다.

## 4. 메뉴와 사용 시점

초기 핵심 메뉴는 다음 다섯 가지다.

1. 오늘의 업무
2. 작업방법
3. 내 일정
4. 중요공지
5. 자주 보는 안내

확인은 출근 전 또는 작업 전, 일정·근무시간 확인, 작업방법을 잠깐 다시 볼 때, 반장·관리자가 안내할 때에 한정한다. 자료를 본 뒤에는 현장 작업으로 돌아가는 흐름을 우선한다.

## 5. 접근성 원칙

- 큰 글자와 큰 터치영역
- 쉬운 문장, 한 문장 한 행동
- 한 화면에 적은 정보와 선택지
- 사진·그림 중심 작업안내
- 복잡한 입력·검색·선택 최소화
- 개인정보와 장애 세부정보의 불필요한 노출 금지

장애명으로 화면을 일괄 결정하지 않는다. 실제 필요한 지원방식에 따라 기본 쉬운 화면, 사진 중심, 큰 글자·고대비, 반장과 함께 사용, 관리자 대리 처리 등을 선택한다.

## 6. 후속 후보

청각장애인용 문장판·전체화면 큰 글자, 진동·화면 중심 알림, 저시력 확대·고대비, 화면 읽기 호환·음성 안내, 그림 중심 의사소통판은 후속 검토 대상이다. 초기 MVP 필수기능으로 확대하지 않는다.

## 7. 검수 기준

- 일반 근로자가 입력 없이 오늘의 업무와 작업방법을 확인할 수 있는가
- 현장 확인과 관리자 기록의 역할이 분리되는가
- 화면이 현장 작업을 방해하거나 상시 휴대전화 사용을 유도하지 않는가
- 개인기기와 공용기기에서 같은 정보가 이해하기 쉽게 보이는가
- 지원방식은 개인 필요에 맞추되 민감정보를 노출하지 않는가
