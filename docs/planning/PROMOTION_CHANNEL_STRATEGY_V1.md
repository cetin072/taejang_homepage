# 홍보 채널 운영 전략 및 콘텐츠 파생 구조 v1

상태: **확정**

이 문서는 태장 홍보팀이 관리하는 채널의 우선순위와 하나의 글감을 여러 채널로 확장하는 기준을 정의합니다.

## 1. 채널 우선순위

1. 홈페이지
2. 네이버 블로그
3. 인스타그램
4. X
5. 유튜브

홈페이지는 태장의 공식 정보, 공지, 주요 성과와 장기 기록을 보존하는 최우선 공식 채널입니다.

## 2. 콘텐츠 제작의 중심

운영 우선순위는 홈페이지가 1순위이지만, 콘텐츠 제작의 중심 원고는 네이버 블로그입니다.

```text
행사·업무 자료 수집
→ 네이버 블로그 상세 원고 작성
→ 홈페이지 공식 기록 작성 또는 외부 링크 카드 등록
→ 인스타그램용 사진·짧은 글 작성
→ X용 핵심 문장·링크 작성
→ 자료가 충분하면 유튜브 기획
```

## 3. 홈페이지 역할

홈페이지는 모든 외부 채널을 통합하고 공식 기록을 남기는 중심 창구입니다.

### 홈페이지 자체 글

- 공식 공지
- 채용공고
- 보도자료
- 주요 행사
- 인증·선정
- 모회사 협력
- 회사의 중요한 성과
- 회사의 공식 입장

### 외부 콘텐츠 카드

- 일상적인 교육
- 작업 현장
- 농장 풍경
- 소규모 활동
- 가벼운 회사소식
- 네이버 블로그·인스타그램·유튜브·X·언론보도 원문

모든 긴 글을 홈페이지에 중복 게시하지 않습니다. 홈페이지는 공식 기록과 통합 아카이브 역할을 하고, 상세한 현장 이야기는 네이버 블로그를 중심으로 운영합니다.

## 4. 채널별 최소 발행 기준

모든 블로그 글을 모든 채널로 강제 확장하지는 않지만, 홍보팀의 운영이 끊기지 않도록 최소 발행량을 둡니다.

| 채널 | 초기 최소 기준 | 비고 |
|---|---:|---|
| 홈페이지 | 주 2건 이상 | 자체 글과 외부 콘텐츠 카드 포함 |
| 네이버 블로그 | 주 3건 이상 | 상세 콘텐츠 중심 원고 |
| 인스타그램 | 주 3건 이상 | 블로그·현장 자료에서 파생 |
| X | 주 3건 이상 | 핵심 소식·링크·현장 기록 |
| 유튜브 | 월 1건 이상, 목표 1~2건 | 행사·활동 자료가 충분할 때 제작 |

월 환산 기준은 홈페이지 8건, 네이버 블로그 12건, 인스타그램 12건, X 12건, 유튜브 1건 이상을 기본 목표로 합니다.

이 수치는 직원 개인평가가 아니라 팀 운영의 누락 방지 기준입니다. 휴일, 행사 취소, 자료 부족 등 합리적 사유가 있으면 팀장이 예외 사유를 기록할 수 있습니다.

## 5. 네이버 블로그 역할

네이버 블로그는 하나의 홍보 주제에서 가장 풍부한 원고를 작성하는 메인 글감 채널입니다.

- 행사 또는 업무의 배경
- 일시와 장소
- 참여자와 관련 기관
- 주요 내용
- 현장 분위기
- 태장에 갖는 의미
- 사진 자료
- 필요 시 향후 계획

## 6. 인스타그램 파생 기준

- 대표 사진 3~10장
- 한 줄 핵심 메시지
- 현장 설명 3~6문장
- 태장의 가치와 연결되는 마무리
- 필요한 해시태그

## 7. X 파생 기준

- 핵심 소식 1~3문장
- 사진 1~4장
- 홈페이지 또는 블로그 원문 링크
- 행사 당일 또는 상세 글 게시 직후 알림

## 8. 유튜브 운영 기준

유튜브는 행사나 활동에서 사진과 영상 자료가 충분히 확보됐을 때 월 1~2개 수준으로 기획·제작합니다.

```text
행사 전 촬영 항목 정리
→ 행사 후 사진·영상 분류
→ 자료 충분성 확인
→ 유튜브 제작 여부 판단
→ 기획안 작성
→ 운영총괄 승인
→ 편집·검토
→ 게시
```

## 9. 플랫폼의 기본 콘텐츠 단위

플랫폼에서는 개별 게시물만 관리하지 않고 하나의 `홍보 주제`를 중심으로 채널별 콘텐츠를 묶어 관리합니다.

```text
홍보 주제: 포장 직무교육
├─ 원본 자료
├─ 네이버 블로그 원고
├─ 홈페이지 게시물 또는 외부 링크 카드
├─ 인스타그램 게시안
├─ X 게시안
└─ 유튜브 기획안 — 필요할 때만
```

## 10. 채널별 진행 상태와 파생 결정

- 작성 중
- 승인 대기
- 보완 필요
- 게시 예정
- 게시 완료
- 제작하지 않음

모든 블로그 글이 반드시 모든 채널로 확장되는 것은 아닙니다. 홍보팀장이 파생 채널을 선택하되, 채널별 최소 발행량이 부족하면 시스템이 부족 수량과 후보 글감을 알려줘야 합니다.

```text
이번 주 채널 운영 현황
홈페이지       2 / 2 완료
네이버 블로그  3 / 3 완료
인스타그램     2 / 3 · 1건 필요
X              1 / 3 · 2건 필요
유튜브         이번 달 0 / 1
```

발행량이 부족한 경우 시스템은 다음 행동을 제안합니다.

- 기존 블로그에서 파생 콘텐츠 만들기
- 기존 사진 자료에서 현장 기록 만들기
- 직원에게 업무 배정
- 홈페이지에 외부 콘텐츠 카드 등록
- 예외 사유 기록

## 11. 초기 지원 기능

초기 버전에서는 AI가 게시물을 자동 완성하는 기능보다 다음을 우선합니다.

- 블로그 제목 불러오기
- 블로그 요약 불러오기
- 대표 사진 선택
- 채널별 작성 양식 제공
- 원문과 파생 콘텐츠 관계 표시
- 채널별 최소 발행 현황 표시
- 부족 수량과 후보 글감 안내

## 12. 개발 검수 기준

- 채널 우선순위가 화면과 업무 흐름에 반영되는가
- 홈페이지가 공식 기록과 통합 아카이브 역할을 하는가
- 네이버 블로그 원고에서 인스타그램과 X 작업을 파생할 수 있는가
- 하나의 홍보 주제 아래 채널별 콘텐츠가 연결되는가
- 유튜브가 자료 충분 시 기획되는가
- 팀장이 파생 여부를 선택할 수 있는가
- 채널별 최소 발행량과 부족 수량을 확인할 수 있는가
- 발행량이 직원 개인평가로 오용되지 않는가
- 합리적인 예외 사유를 기록할 수 있는가

## 13. 확정 문장

> 태장 홍보는 홈페이지를 최우선 공식 채널로 관리하고, 네이버 블로그를 상세 콘텐츠의 중심 원고로 사용한다. 블로그 글을 기반으로 인스타그램과 X 콘텐츠를 각 채널 특성에 맞게 재구성한다. 모든 글을 모든 채널에 강제 확장하지는 않지만 채널별 최소 발행량을 팀 운영 기준으로 관리한다. 유튜브는 행사 등에서 충분한 자료가 확보된 경우 월 1~2개 수준으로 제작한다. 플랫폼에서는 하나의 홍보 주제와 파생 콘텐츠를 하나의 묶음으로 관리한다.
