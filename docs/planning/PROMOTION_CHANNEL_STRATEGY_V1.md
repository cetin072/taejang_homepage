# 홍보 채널 운영 전략 및 콘텐츠 파생 구조 v1

상태: **확정**

이 문서는 태장 홍보팀이 관리하는 채널의 우선순위와 하나의 글감을 여러 채널로 확장하는 기준을 정의합니다.

## 1. 채널 우선순위

태장 홍보팀의 채널 우선순위는 다음과 같습니다.

1. 홈페이지
2. 네이버 블로그
3. 인스타그램
4. X
5. 유튜브

홈페이지는 태장의 공식 정보, 공지, 주요 성과와 장기 기록을 보존하는 최우선 공식 채널입니다.

## 2. 콘텐츠 제작의 중심

운영 우선순위는 홈페이지가 1순위이지만, 콘텐츠 제작의 중심 원고는 네이버 블로그입니다.

네이버 블로그에서 행사·업무·교육·현장 활동을 충분히 설명한 상세 원고를 만든 뒤, 그 내용을 바탕으로 인스타그램과 X용 콘텐츠를 채널 특성에 맞게 재구성합니다.

기본 흐름은 다음과 같습니다.

```text
행사·업무 자료 수집
→ 네이버 블로그 상세 원고 작성
→ 홈페이지 공식 기록 작성 또는 외부 링크 카드 등록
→ 인스타그램용 사진·짧은 글 작성
→ X용 핵심 문장·링크 작성
→ 자료가 충분하면 유튜브 기획
```

## 3. 홈페이지 역할

홈페이지는 모든 외부 채널을 통합하고 공식 기록을 남기는 중심 창구입니다.

### 홈페이지 자체 글로 작성할 대상

- 공식 공지
- 채용공고
- 보도자료
- 주요 행사
- 인증·선정
- 모회사 협력
- 회사의 중요한 성과
- 회사의 공식 입장

### 외부 콘텐츠 카드로 연결할 대상

- 일상적인 교육
- 작업 현장
- 농장 풍경
- 소규모 활동
- 가벼운 회사소식
- 네이버 블로그·인스타그램·유튜브·X·언론보도 원문

모든 긴 글을 홈페이지에 중복 게시하지 않습니다. 홈페이지는 공식 기록과 통합 아카이브 역할을 하고, 상세한 현장 이야기는 네이버 블로그를 중심으로 운영합니다.

## 4. 네이버 블로그 역할

네이버 블로그는 하나의 홍보 주제에서 가장 풍부한 원고를 작성하는 메인 글감 채널입니다.

기본 구성은 다음을 포함합니다.

- 행사 또는 업무의 배경
- 일시와 장소
- 참여자와 관련 기관
- 주요 내용
- 현장 분위기
- 태장에 갖는 의미
- 사진 자료
- 필요 시 향후 계획

블로그 글은 인스타그램과 X 콘텐츠를 만드는 기준 원고로 사용합니다.

## 5. 인스타그램 파생 기준

인스타그램은 블로그 원고를 단순 축약하지 않고 사진 중심으로 재구성합니다.

- 대표 사진 3~10장
- 한 줄 핵심 메시지
- 현장 설명 3~6문장
- 태장의 가치와 연결되는 마무리
- 필요한 해시태그

블로그 글에서 사진과 핵심 메시지를 선별해 인스타그램용 게시안을 만듭니다.

## 6. X 파생 기준

X는 짧고 즉각적인 소식 전달에 사용합니다.

- 핵심 소식 1~3문장
- 사진 1~4장
- 홈페이지 또는 블로그 원문 링크
- 행사 당일 또는 상세 글 게시 직후 알림

X 글을 독립적으로 처음부터 기획하기보다 블로그 원고의 핵심 문장을 바탕으로 작성합니다.

## 7. 유튜브 운영 기준

유튜브는 월간 수량을 억지로 채우는 방식으로 운영하지 않습니다.

행사나 활동에서 사진과 영상 자료가 충분히 확보됐을 때 월 1~2개 수준으로 기획·제작합니다.

적합한 소재는 다음과 같습니다.

- 개소식
- 직무교육
- 농장 활동
- 사회공헌 행사
- 모회사·기관 방문
- 제품 제작 과정
- 문화예술 활동
- 월간·분기 활동 모음

기본 흐름은 다음과 같습니다.

```text
행사 전 촬영 항목 정리
→ 행사 후 사진·영상 분류
→ 자료 충분성 확인
→ 유튜브 제작 여부 판단
→ 기획안 작성
→ 운영총괄 승인
→ 편집·검토
→ 게시
```

## 8. 플랫폼의 기본 콘텐츠 단위

플랫폼에서는 개별 게시물만 관리하지 않고 하나의 `홍보 주제`를 중심으로 채널별 콘텐츠를 묶어 관리합니다.

예시:

```text
홍보 주제: 포장 직무교육
├─ 원본 자료
│  ├─ 사진
│  ├─ 영상
│  └─ 행사 정보
├─ 네이버 블로그 원고
├─ 홈페이지 게시물 또는 외부 링크 카드
├─ 인스타그램 게시안
├─ X 게시안
└─ 유튜브 기획안 — 필요할 때만
```

## 9. 채널별 진행 상태

하나의 홍보 주제 화면에서 채널별 상태를 확인할 수 있어야 합니다.

- 작성 중
- 승인 대기
- 보완 필요
- 게시 예정
- 게시 완료
- 제작하지 않음

모든 블로그 글이 반드시 모든 채널로 확장되는 것은 아닙니다. 홍보팀장이 다음 중 하나를 선택할 수 있어야 합니다.

- 인스타그램 제작
- X 제작
- 두 채널 모두 제작
- 추가 제작하지 않음
- 유튜브 기획 검토

## 10. 파생 업무 제안

네이버 블로그 글이 승인되거나 게시되면 시스템은 다음 후속 업무를 제안할 수 있습니다.

- 인스타그램 글 만들기
- X 글 만들기
- 홈페이지에 연결하기
- 행사 자료가 충분한 경우 유튜브 기획 검토

초기 버전에서는 AI가 게시물을 완성하는 기능보다 다음을 우선합니다.

- 블로그 제목 불러오기
- 블로그 요약 불러오기
- 대표 사진 선택
- 채널별 작성 양식 제공
- 원문과 파생 콘텐츠 관계 표시

## 11. 개발 검수 기준

홍보 모듈 구현 시 다음을 확인합니다.

- 채널 우선순위가 화면과 업무 흐름에 반영돼 있는가
- 홈페이지가 공식 기록과 통합 아카이브 역할을 하는가
- 네이버 블로그 원고에서 인스타그램과 X 작업을 파생할 수 있는가
- 하나의 홍보 주제 아래 채널별 콘텐츠가 연결되는가
- 유튜브가 필수 일일 작업이 아니라 자료 충분 시 기획되는가
- 모든 채널 제작을 직원에게 강제하지 않고 팀장이 제작 여부를 선택할 수 있는가
- 원문과 파생 게시물의 상태와 관계를 추적할 수 있는가

## 12. 확정 문장

> 태장 홍보는 홈페이지를 최우선 공식 채널로 관리하고, 네이버 블로그를 상세 콘텐츠의 중심 원고로 사용한다. 블로그 글을 기반으로 인스타그램과 X 콘텐츠를 각 채널 특성에 맞게 재구성한다. 유튜브는 행사 등에서 충분한 사진과 영상 자료가 확보된 경우 월 1~2개 수준으로 기획·제작한다. 플랫폼에서는 하나의 홍보 주제와 그 주제에서 파생된 채널별 콘텐츠를 하나의 묶음으로 관리한다.
