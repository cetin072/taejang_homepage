# 결재·승인·대표 상신 체계 v1

상태: **확정**

## 1. 기본 원칙

태장 업무 플랫폼의 기본 최종 승인자는 **총괄 등기이사**로 둡니다.

일반적인 회사 운영 안건은 총괄 등기이사의 승인으로 종결하며, 모든 안건을 대표이사에게 자동 상신하지 않습니다.

대표이사 승인이 필요하다고 총괄 등기이사가 판단한 안건만 선별하여 대표이사에게 올립니다.

## 2. 기본 승인 흐름

### 일반 부서 업무

담당자 작성 또는 직원 업무 완료 → 팀장 검토 → 총괄 등기이사 최종 승인 → 완료

### 팀장 선에서 종결 가능한 단순 업무

직원 업무 완료 → 팀장 확인 → 완료

단, 회사 공통 운영, 비용, 인력, 대외 협의, 부서 간 조정이 포함되면 총괄 승인 대상으로 올립니다.

### 대표이사 상신 안건

담당 부서 또는 팀장 작성 → 총괄 등기이사 검토·정리 → 총괄이 대표 승인 필요 여부 판단 → 대표이사 상신 → 대표이사 결정

## 3. 총괄 등기이사의 역할

총괄 등기이사는 다음을 판단합니다.

- 해당 안건을 본인이 최종 승인할 수 있는지
- 대표이사의 경영 판단이 필요한지
- 부서 의견, 비용, 일정, 위험이 충분히 정리됐는지
- 대표이사에게 올리기 전에 추가 보완이 필요한지
- 대표이사에게 어떤 선택지와 핵심정보를 보여줄지

총괄은 단순 전달자가 아니라 기본 최종 승인자이자 대표 상신 여부를 판단하는 관문 역할을 합니다.

## 4. 대표이사에게 상신할 수 있는 안건 예시

다음은 자동 상신 항목이 아니라, 총괄이 중요도와 영향을 검토하여 대표 승인이 필요하다고 판단할 수 있는 대표적 사례입니다.

- 중요 계약과 협약
- 주요 인사와 조직 변경
- 고액 또는 중요한 지출
- 신규사업 착수·중단
- 모회사·공공기관 등 주요 대외 협의
- 회사 명의의 중요한 공식 입장
- 중대한 사고, 분쟁, 법적 위험
- 회사 전체 방향이나 장기 전략에 영향을 주는 안건

## 5. 대표이사 화면 원칙

대표이사에게는 총괄이 선별한 안건만 표시합니다.

한 안건에는 다음을 우선 표시합니다.

- 안건 제목
- 결정이 필요한 이유
- 총괄 의견
- 선택지
- 비용·일정·위험
- 관련 자료
- 결정기한

대표이사는 승인, 보완, 보류, 반려 중 하나를 선택하고 필요한 경우 짧은 의견을 남깁니다.

## 6. 플랫폼 구현 원칙

- 기본 최종 승인자 필드는 총괄 등기이사로 설정
- 총괄 승인 시 일반 안건은 완료 처리 가능
- 총괄 화면에 `대표 승인 필요` 또는 `대표에게 상신` 기능 제공
- 대표 상신 전 총괄 의견과 핵심 요약 입력
- 대표 화면에는 상신된 안건만 노출
- 팀장·담당자가 대표이사에게 직접 상신하는 경로는 기본적으로 제공하지 않음
- 긴급상황은 총괄에게 즉시 보고하고, 필요 시 총괄이 대표에게 긴급 상신

## 7. 초기 개발 범위

초기에는 다음 기능만 구현합니다.

- 팀장 검토
- 총괄 최종 승인
- 총괄의 대표 상신 판단
- 대표 승인·보완·보류·반려
- 처리 의견
- 관련 자료 링크
- 처리기한
- 승인 이력
- 역할별 대기 목록

복잡한 다단계 전자결재, 병렬결재, 합의·협조·참조 구분은 초기 범위에서 제외합니다.
