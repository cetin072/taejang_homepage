# 태장 조직 및 역할·보고 체계 v1

상태: **확정**

이 문서는 태장 업무 플랫폼과 각 부서 모듈의 권한·보고·승인 구조를 설계하기 위한 현재 조직 기준입니다.

## 1. 경영진

### 대표이사 이영희

- 회사 최종 의사결정
- 대외 대표
- 중대한 경영·고용·공식 발표 승인
- 운영총괄이 선별해 올린 핵심 안건 최종 판단
- 일반 부서 실무와 일상 승인에는 직접 개입하지 않음
- 필요할 때는 모든 부서, 업무, 승인, 기록과 관련 기능에 접근해 세부 내용을 확인할 수 있음
- 기본 대시보드는 최종 승인, 핵심 경영지표와 중대한 예외사항 중심으로 가장 간결하게 구성

### 총괄운영 등기이사 김형철

- 회사 운영 전반 총괄
- 부서 간 업무 조정
- 중요 승인과 문제 해결
- 홍보·물류·근로자지원·생산 등 각 기능의 운영 현황 관리
- 대표이사에게 올릴 안건 선별
- 팀장 지시, 성과·위험 관리
- 태장 업무 플랫폼의 최상위 실무 관리자
- 필요할 때는 모든 부서, 업무, 승인, 기록과 관련 기능에 접근해 세부 내용을 확인할 수 있음
- 기본 대시보드는 오늘 처리할 일, 중요 승인, 지연·위험과 부서 간 조정사항 중심으로 요약 구성

### 사내이사 이선영

- 대외업무 보조
- 대관업무 보조
- 모회사 담당자 관리 지원
- 비정기 출근
- 일상 부서 운영과 상시 승인 담당자는 아님
- 플랫폼에는 실제 담당 범위에 맞는 제한적 조회·보고·업무 권한을 부여

## 2. 주요 부서와 책임자

### 홍보팀

책임자: 이환호 홍보팀장

핵심 역할:

- 홈페이지와 공식 콘텐츠 운영
- 네이버 블로그·인스타그램·X·유튜브 관리
- 모회사 추가 유치에 필요한 홍보 근거 제작
- 태장 이미지와 신뢰도 향상
- 상품·서비스 매출 지원
- 행사·교육·생산·사회공헌 기록

장애인 근로자 배치:

- 오전 2명, 오전 하위 팀장 1명
- 오후 2명, 오후 하위 팀장 1명

### 물류팀

책임자: 황성화 물류팀장

핵심 역할:

- 입고·출고
- 재고 관리
- 발주
- 포장 준비와 배송 확인
- 재고 부족과 납기 위험 보고

장애인 근로자 배치:

- 오전 2명, 오전 하위 팀장 1명
- 오후 2명, 오후 하위 팀장 1명

### 생산팀

현재 상태:

- 팀장 공석
- 장애인 근로자 오전 5명·오후 5명
- 민화작가는 주 2회 현장 지도하는 외부 지도자
- 현장 상시 운영은 총괄반장, 업무반장, 현장 관리자와 공통관리 체계가 담당

민화작가는 작업 방향·작품 지도를 제공하되 직원·팀장 권한이나 상시 현장관리 권한을 갖지 않습니다.

### 초기 적응 지원

- 별도 전담 조직으로 고정하지 않고 근로자지원 담당자와 현장관리자가 필요할 때 지원
- 배치·적응·관찰은 민감정보를 최소화해 관리
- 일반 근로자 화면에는 지원 경과가 아니라 필요한 업무안내만 제공

### 근로자지원팀

현재 상태:

- 책임 팀장 모집 중
- 견습 1명 있으나 미확정
- 향후 1명에서 2명 체계 검토

핵심 역할:

- 입사 전 면접 동석과 초기 기록
- 신규 근로자 적응 확인
- 상담과 특이사항 기록
- 갈등·부적응·퇴사 위험 조기 발견
- 직무·근무시간·지원 조정 제안
- 보호자·기관 연락
- 퇴사 방지와 고용 유지
- 퇴사 시 구인·면접·신규 입사 연결

### 근로지원인

- 1명 추가 예정
- 작업 중 직접적인 수행 지원
- 의사소통·작업 순서·이동·도구 사용 보조
- 현장에서 발견한 어려움을 담당자에게 보고

## 3. 장애인 근로자 운영 구조

- 1차 최종 채용 목표: 장애인 근로자 25명
- 부서·오전·오후 배치는 별도 인원 배치표에서 관리
- 개인의 장애 유형과 정도는 조직도나 일반 화면에 표시하지 않습니다.
- 실제 지원방식이 필요한 담당자만 최소 범위로 접근합니다.

현재 인원과 부서별 배치는 `EXPECTED_STAFFING_V1.md`, 일반 근로자 화면은 `GENERAL_WORKER_INFORMATION_BOARD_V1.md`를 우선합니다.

## 4. 반장과 하위 팀장

### 총괄반장·업무반장·현장 관리자

- 총괄반장: 공통 안내, 이동·휴식·일정과 긴급 전달
- 업무반장: 작업 순서 안내, 현장 확인, 작업방법 자료 작성·수정
- 현장 관리자: 작업반 운영, 예외 변경, 최소 기록과 인계
- 필요 시 근로자지원 담당자가 적응·지원 협의에 연결

현장 책임자는 실제 배정·진행·완료를 확인하고 필요한 내용만 관리자 화면에 기록합니다. 일반 근로자는 읽기·확인 중심이며 시작·완료·실적을 입력하지 않습니다. 현장 역할은 겸임할 수 있으나 인사결정과 상담 원문 열람 권한은 부여하지 않습니다.

## 5. 보고 체계

### 일반 업무

근로자 → 업무반장 또는 현장 관리자 → 부서 팀장 → 운영총괄 → 필요 시 대표이사

### 출근·공통생활 문제

근로자 → 총괄반장 또는 현장 관리자 → 근로지원인·담당 팀장 → 필요 시 근로자지원팀

### 상담·갈등·부적응·퇴사 위험

근로자 또는 주변 직원 → 근로자지원팀 → 필요한 조치만 부서 팀장과 협의 → 중요한 고용 위험은 운영총괄 보고 → 중대한 경우 대표이사 보고

### 긴급 안전 문제

발견자 → 현장 책임자와 운영총괄 즉시 보고 → 안전 확보 → 필요한 기관·보호자 연락

일반 계급 순서를 기다리지 않습니다.

## 6. 플랫폼 역할 구분

초기 역할은 다음과 같이 구분합니다.

1. 대표이사
2. 운영총괄
3. 시스템 최고관리자
4. 부서 팀장
5. 총괄반장·업무반장·현장 관리자
6. 근로자지원 책임자
7. 근로자지원 담당자
8. 홍보팀장
9. 홍보 직원
10. 일반 사무직원
11. 일반 장애인 근로자
12. 근로지원인
13. 외부지도자

사내이사 등 법적 직책은 조직 표시와 보고체계에 사용하되 실제 시스템 권한은 위 역할과 부서·작업반 범위를 별도로 배정한다. 한 사용자는 여러 역할을 가질 수 있다.

현장 반장·관리자에게는 자신의 작업반 배정, 현장 확인, 최소 기록, 공지·작업방법 관리와 지원 연결만 허용합니다. 상담 원문이나 민감 인사정보에는 접근할 수 없습니다.

### 대표이사·운영총괄의 전사 접근과 대시보드 원칙

- 대표이사와 운영총괄은 본인이 필요하다고 판단하면 모든 부서의 업무, 승인, 기록과 첨부자료를 조회할 수 있습니다. 계정 상태·역할 변경은 별도 시스템 최고관리자 역할이 있어야 합니다.
- 기본 대시보드에 표시되지 않는다는 이유로 접근권한이 제한되어서는 안 됩니다.
- 세부 자료는 부서별 메뉴, 전체 메뉴, 통합검색과 기록 조회를 통해 찾아볼 수 있어야 합니다.
- 직급이 올라갈수록 열람 범위는 넓어지지만, 기본 대시보드에 노출되는 정보는 더 압축합니다.
- 운영총괄 대시보드는 오늘 처리할 일, 중요 승인, 지연·위험, 부서 간 조정과 대표이사 보고 필요사항을 중심으로 구성합니다.
- 대표이사 대시보드는 최종 승인, 핵심 경영지표, 중대한 인사·고용·예산·계약 안건과 예외사항을 중심으로 운영총괄보다 더 간결하게 구성합니다.
- 상담 원문 등 제한정보는 전사 직급만으로 자동 공개하지 않으며 업무상 필요성과 감사기록 원칙을 적용합니다.

## 7. 문서 상태와 개발 검수 기준

현장 운영·일반 근로자 화면에 관한 과거 입력 중심 설명은 `GENERAL_WORKER_INFORMATION_BOARD_V1.md`와 `FIELD_LEAD_OPERATION_RECORDS_V1.md` 기준으로 대체합니다.

## 8. 개발 검수 기준

- 법적 직책과 실제 업무 권한이 구분돼 있는가
- 대표이사와 운영총괄이 필요할 때 전사 세부 정보와 기능에 접근할 수 있는가
- 대표이사 대시보드는 최종 승인과 핵심 경영정보 중심으로 가장 간결한가
- 운영총괄 대시보드는 전사 운영, 중요 승인, 지연·위험과 조정사항 중심으로 요약되는가
- 대시보드 미노출과 접근권한 제한을 혼동하지 않는가
- 이선영 사내이사의 비정기 대외·대관 보조 역할이 반영되는가
- 민화작가의 주 2회 전문 지도와 상시 현장 관리가 구분되는가
- 초보 직무적응반이 별도 전담자 없이 공통관리 중인 현재 상태를 반영하는가
- 반장과 하위 팀장의 제한적 권한이 구분되는가
- 상담·장애·보호자 정보가 최소 권한과 감사기록으로 보호되는가
