# 태장 조직 및 역할·보고 체계 v1

상태: **확정**

이 문서는 태장 업무 플랫폼과 각 부서 모듈의 권한·보고·승인 구조를 설계하기 위한 현재 조직 기준입니다.

## 1. 경영진

### 대표이사 이영희

- 회사 최종 의사결정
- 대외 대표
- 중대한 경영·고용·공식 발표 승인
- 운영총괄이 선별해 올린 핵심 안건 최종 판단
- 일반 부서 실무와 일상 승인에는 직접 개입하지 않음
- 필요할 때는 모든 부서, 업무, 승인, 기록과 관련 기능에 접근해 세부 내용을 확인할 수 있음
- 기본 대시보드는 최종 승인, 핵심 경영지표와 중대한 예외사항 중심으로 가장 간결하게 구성

### 총괄운영 등기이사 김형철

- 회사 운영 전반 총괄
- 부서 간 업무 조정
- 중요 승인과 문제 해결
- 홍보·물류·근로자지원·생산 등 각 기능의 운영 현황 관리
- 대표이사에게 올릴 안건 선별
- 팀장 지시, 성과·위험 관리
- 태장 업무 플랫폼의 최상위 실무 관리자
- 필요할 때는 모든 부서, 업무, 승인, 기록과 관련 기능에 접근해 세부 내용을 확인할 수 있음
- 기본 대시보드는 오늘 처리할 일, 중요 승인, 지연·위험과 부서 간 조정사항 중심으로 요약 구성

### 사내이사 이선영

- 대외업무 보조
- 대관업무 보조
- 모회사 담당자 관리 지원
- 비정기 출근
- 일상 부서 운영과 상시 승인 담당자는 아님
- 플랫폼에는 실제 담당 범위에 맞는 제한적 조회·보고·업무 권한을 부여

## 2. 주요 부서와 책임자

### 홍보팀

책임자: 이환호 홍보팀장

핵심 역할:

- 홈페이지와 공식 콘텐츠 운영
- 네이버 블로그·인스타그램·X·유튜브 관리
- 모회사 추가 유치에 필요한 홍보 근거 제작
- 태장 이미지와 신뢰도 향상
- 상품·서비스 매출 지원
- 행사·교육·생산·사회공헌 기록

장애인 근로자 배치:

- 오전 2명, 오전 하위 팀장 1명
- 오후 2명, 오후 하위 팀장 1명

### 물류팀

책임자: 황성화 물류팀장

핵심 역할:

- 입고·출고
- 재고 관리
- 발주
- 포장 준비와 배송 확인
- 재고 부족과 납기 위험 보고

장애인 근로자 배치:

- 오전 2명, 오전 하위 팀장 1명
- 오후 2명, 오후 하위 팀장 1명

### 민화팀

상위 담당자: 민화작가

운영 형태:

- 민화작가 주 2회 출근
- 작품 지도, 교육, 작업 방향 제시
- 상시 현장 관리는 반장·하위 팀장·공통관리 체계와 연결

장애인 근로자 배치:

- 오전 4~5명, 오전 하위 팀장 1명
- 오후 4~5명, 오후 하위 팀장 1명

### 초보 직무적응반

- 별도 상위 전담 책임자 없음
- 현재 공통관리 중
- 신입·초기 적응 근로자의 업무 이해와 직무 탐색 지원
- 홍보·물류·민화 등 정규 직무팀으로 이동할 수 있도록 기초 적응 관리
- 향후 근로자지원 모듈과 연계해 배치·적응·관찰 기록을 관리

### 근로자지원팀

현재 상태:

- 책임 팀장 모집 중
- 견습 1명 있으나 미확정
- 향후 1명에서 2명 체계 검토

핵심 역할:

- 입사 전 면접 동석과 초기 기록
- 신규 근로자 적응 확인
- 상담과 특이사항 기록
- 갈등·부적응·퇴사 위험 조기 발견
- 직무·근무시간·지원 조정 제안
- 보호자·기관 연락
- 퇴사 방지와 고용 유지
- 퇴사 시 구인·면접·신규 입사 연결

### 근로지원인

- 1명 추가 예정
- 작업 중 직접적인 수행 지원
- 의사소통·작업 순서·이동·도구 사용 보조
- 현장에서 발견한 어려움을 담당자에게 보고

## 3. 장애인 근로자 운영 구조

- 오전반 12명
- 오후반 12명
- 총 24명
- 경증 3명, 나머지는 중증
- 장애 유형은 지체·지적·정신·자폐 등 다양

개인의 장애 유형과 정도는 조직도나 일반 화면에 표시하지 않습니다. 지원에 꼭 필요한 사람만 최소 범위로 접근합니다.

## 4. 반장과 하위 팀장

### 오전·오후 반장

각 반마다 반장 1명을 둡니다.

- 해당 시간대 전체 근로자 공통 안내
- 출근·이동·휴식·공통 일정 확인 지원
- 긴급 상황 전달
- 직무팀 사이 연결
- 특이사항을 담당자에게 알림

### 직무별 하위 팀장

- 자기 팀의 작업 순서 안내
- 업무 진행과 완료 여부 확인 지원
- 팀원의 어려움을 상위 담당자에게 전달
- 일반 인사관리자가 아니라 현장 업무 리더

반장과 하위 팀장은 겸임할 수 있으나 플랫폼 권한은 역할별로 구분합니다.

## 5. 보고 체계

### 일반 업무

근로자 → 직무별 하위 팀장 → 부서 팀장 또는 담당자 → 운영총괄 → 필요 시 대표이사

### 출근·공통생활 문제

근로자 → 오전·오후 반장 → 근로지원인 또는 담당 팀장 → 필요 시 근로자지원팀

### 상담·갈등·부적응·퇴사 위험

근로자 또는 주변 직원 → 근로자지원팀 → 필요한 조치만 부서 팀장과 협의 → 중요한 고용 위험은 운영총괄 보고 → 중대한 경우 대표이사 보고

### 긴급 안전 문제

발견자 → 현장 책임자와 운영총괄 즉시 보고 → 안전 확보 → 필요한 기관·보호자 연락

일반 계급 순서를 기다리지 않습니다.

## 6. 플랫폼 역할 구분

초기 역할은 다음과 같이 구분합니다.

1. 대표이사
2. 운영총괄
3. 사내이사
4. 부서 팀장
5. 근로자지원팀 책임자
6. 근로자지원 담당자
7. 근로지원인
8. 오전·오후 반장
9. 직무별 하위 팀장
10. 일반 근로자

반장과 하위 팀장에게는 자신의 반·팀 업무 확인, 완료 표시 보조, 어려움 보고, 공지 확인, 지원 요청 정도만 허용합니다. 상담 원문이나 민감 인사정보에는 접근할 수 없습니다.

### 대표이사·운영총괄의 전사 접근과 대시보드 원칙

- 대표이사와 운영총괄은 본인이 필요하다고 판단하면 모든 부서의 업무, 승인, 기록, 첨부자료와 관련 기능에 접근할 수 있습니다.
- 기본 대시보드에 표시되지 않는다는 이유로 접근권한이 제한되어서는 안 됩니다.
- 세부 자료는 부서별 메뉴, 전체 메뉴, 통합검색과 기록 조회를 통해 찾아볼 수 있어야 합니다.
- 직급이 올라갈수록 열람 범위는 넓어지지만, 기본 대시보드에 노출되는 정보는 더 압축합니다.
- 운영총괄 대시보드는 오늘 처리할 일, 중요 승인, 지연·위험, 부서 간 조정과 대표이사 보고 필요사항을 중심으로 구성합니다.
- 대표이사 대시보드는 최종 승인, 핵심 경영지표, 중대한 인사·고용·예산·계약 안건과 예외사항을 중심으로 운영총괄보다 더 간결하게 구성합니다.
- 전사 접근권한이 있더라도 민감정보 열람은 업무상 필요성과 감사기록 원칙을 적용합니다.

## 7. 개발 검수 기준

- 법적 직책과 실제 업무 권한이 구분돼 있는가
- 대표이사와 운영총괄이 필요할 때 전사 세부 정보와 기능에 접근할 수 있는가
- 대표이사 대시보드는 최종 승인과 핵심 경영정보 중심으로 가장 간결한가
- 운영총괄 대시보드는 전사 운영, 중요 승인, 지연·위험과 조정사항 중심으로 요약되는가
- 대시보드 미노출과 접근권한 제한을 혼동하지 않는가
- 이선영 사내이사의 비정기 대외·대관 보조 역할이 반영되는가
- 민화작가의 주 2회 전문 지도와 상시 현장 관리가 구분되는가
- 초보 직무적응반이 별도 전담자 없이 공통관리 중인 현재 상태를 반영하는가
- 반장과 하위 팀장의 제한적 권한이 구분되는가
- 상담·장애·보호자 정보가 최소 권한과 감사기록으로 보호되는가
