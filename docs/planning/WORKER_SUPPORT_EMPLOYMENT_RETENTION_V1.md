# 근로자지원·고용유지 모듈 v1 확정 기획

## 1. 문서 목적

이 문서는 태장 업무 플랫폼의 `근로자지원·고용유지` 모듈에 대한 1차 확정 기획이다.

이 모듈의 목적은 장애인 근로자의 심리적 안정, 직무 적응, 회사생활 지원, 근무환경 및 복리후생 개선, 갈등·결근·퇴사위험의 조기 발견과 대응을 통해 장기근속과 안정적인 고용유지를 지원하는 것이다. 이는 태장과 모회사의 장애인 완전고용 유지라는 핵심 목표와 직접 연결된다.

초기 버전은 세분화보다 직관성과 편의성을 우선하며, 실제 운영 중 필요성이 확인된 기능만 추가한다.

## 2. 부서 명칭과 역할

- 공식 부서명: `근로자지원팀`
- 플랫폼 모듈명: `근로자지원·고용유지`

### 2.1 인사 기능

인사 기능은 회사와 근로자 사이의 공식 고용관계를 관리한다.

주요 범위:

- 채용과 입사
- 근로계약과 근로조건
- 소속 부서, 직무, 근무시간의 공식 결정과 변경
- 휴직, 복직, 퇴사, 재직 상태
- 공식 인사정보와 인사서류
- 공식 인사조치와 변경이력

현재 태장에는 별도 인사부서가 없으므로 대표이사와 운영총괄이 인사 기능을 수행한다. 플랫폼에서는 향후 인사담당자에게 권한을 위임할 수 있도록 인사 권한과 근로자지원 권한을 처음부터 분리한다.

### 2.2 근로자지원팀

근로자지원팀은 근로자가 안정적으로 근무하고 장기근속할 수 있도록 상담, 관찰, 지원, 후속 확인과 조정 제안을 담당한다.

주요 범위:

- 상담과 현장 관찰
- 직무·회사생활 적응 확인
- 근무환경과 복리후생 관련 지원
- 갈등, 반복 결근, 출근 거부, 퇴사위험 등 고용유지 위험의 조기 발견
- 보호자·외부기관과의 협의 필요성 확인
- 직무, 근무시간, 부서 조정 제안
- 지원조치 후 효과 확인

근로자지원팀은 공식 인사결정을 하지 않는다. 문제를 발견하고 지원 방향을 제안할 수 있으나, 공식 직무·부서·근무시간 변경 등은 인사 기능에서 결정하고 처리한다.

## 3. 인사카드 구조

인사카드는 하나의 화면으로 제공하되 데이터와 권한은 구분한다. 초기 화면은 3개 탭만 사용한다.

### 3.1 상단 요약

- 사진과 이름
- 사원번호
- 소속 부서
- 담당 직무
- 근무시간
- 입사일
- 재직 상태
- 현재 진행 중인 지원 건

### 3.2 탭 1: 기본·근무정보

- 연락처와 주소
- 비상연락처
- 보호자·지원기관 연락처
- 소속 부서와 담당 직무
- 근무시간
- 입사일과 재직 상태
- 부서·직무·근무시간의 공식 변경이력

현재 정보는 바로 보여주고 과거 이력은 필요할 때 펼쳐본다.

### 3.3 탭 2: 상담·지원 기록

- 상담일지
- 현장 관찰기록
- 지원 요청과 지원조치
- 후속 확인
- 보호자·기관 연락기록
- 직무·근무시간 조정 제안
- 특이사항 보고와 처리 결과

기록은 날짜순으로 누적한다.

### 3.4 탭 3: 서류·이력

- 이력서
- 근로계약서
- 입사자료와 증빙자료
- 공식 인사정보 변경이력
- 주요 열람·수정 이력

열람이력 등 관리용 정보는 기본 화면에서 강조하지 않고 필요할 때만 확인한다.

## 4. 권한 원칙

### 4.1 근로자지원 담당자

- 담당 근로자의 기본·근무정보 열람
- 상담·관찰·지원기록 작성과 후속 확인
- 공식 인사정보 직접 수정 불가
- 인사정보 오류 발견 시 수정 요청 가능
- 급여, 계좌, 세금 등 업무와 무관한 민감정보 접근 불가

### 4.2 근로자지원팀장

- 근로자지원팀 전체 기록 열람
- 담당자 배정과 기록 검토
- 특이사항 보고 여부 판단
- 부서 협의 필요사항 확인

### 4.3 운영총괄

- 전체 인사·근로자지원 정보 접근 가능
- 기본 대시보드에는 세부 상담기록을 노출하지 않음
- 특이사항과 조정 필요 건을 중심으로 확인

### 4.4 대표이사

- 전체 정보 접근 가능
- 중대한 고용유지, 인사, 안전, 예산, 운영결정 중심으로 확인
- 기본 대시보드는 핵심 승인과 최종 판단사항 위주로 구성

### 4.5 다른 부서 팀장

- 상담 원문과 개인적인 민감정보는 열람하지 않음
- 자신의 부서에서 조치해야 할 업무상 지원사항만 확인
- 실행방법과 담당자를 정하고 결과를 회신

## 5. 상담·지원 기록 작성

상담·지원 기록은 `기록 추가` 버튼 하나로 작성한다.

기본 항목:

- 기록일시
- 대상 근로자
- 기록 유형
- 주요 내용
- 필요한 지원 또는 조치
- 후속 확인일
- 특이사항 여부

초기 기록 유형:

- 상담
- 현장 관찰
- 지원조치
- 보호자·기관 연락
- 후속 확인
- 특이사항 보고

## 6. 부서 간 협의 원칙

근로자지원팀은 다른 부서에 직접 업무지시를 하지 않는다.

- 근로자지원팀은 확인된 어려움과 필요한 지원 방향을 전달하고 협의를 요청한다.
- 해당 부서 팀장은 자신의 부서 내 실행방법과 담당자를 결정한다.
- 상담 원문이 아니라 업무 수행에 필요한 지원사항만 공유한다.
- 부서 간 의견 충돌이나 중요한 운영조정이 필요한 경우 운영총괄이 최종 결정한다.

상태값, 처리기한, 회신 양식 등 세부 절차는 실제 운영하면서 필요성을 확인한 뒤 추가한다.

## 7. 운영총괄 특이사항 보고

일반 상담·관찰·지원 기록은 운영총괄 대시보드에 표시하지 않는다. 평상시에는 관련 카드나 알림이 보이지 않는 것이 정상 상태다.

다음과 같이 운영총괄의 확인 또는 결정이 필요한 경우에만 특이사항으로 보고한다.

- 퇴사 의사, 출근 거부, 반복 결근·지각
- 중대한 갈등, 괴롭힘, 부당대우 의심
- 건강·안전 우려
- 직무·근무시간·부서 변경 검토 필요
- 보호자·기관과 공식 협의 필요
- 반복 지원에도 개선되지 않음
- 해당 부서와 협의가 되지 않음
- 예산, 인력, 운영방침 결정 필요
- 담당자가 단독으로 판단하기 어려운 상황

대시보드에는 긴 상담내용이 아니라 다음만 표시한다.

- 대상 근로자
- 특이사항 유형
- 핵심 요약
- 현재까지의 조치
- 운영총괄이 확인하거나 결정할 사항
- 긴급도와 처리기한

초기 상태값은 `확인 필요`, `조치 중`, `완료`로 단순화한다.

## 8. 근로자별 기록과 일일보고

근로자별 상담·지원 기록을 기본 데이터로 사용한다.

일일보고서는 그날 작성된 기록을 바탕으로 다음을 자동 집계한다.

- 상담 건수
- 현장 관찰 건수
- 지원조치 건수
- 후속 확인 예정 건수
- 특이사항 건수

초기에는 AI 자연어 요약을 사용하지 않는다. 담당자가 같은 내용을 반복 작성하지 않도록 자동 집계 중심으로 시작하고, 자연어 요약은 실제 기록량과 필요성이 확인된 뒤 검토한다.

## 9. 근로자지원 담당자 메인 화면

메인 화면은 통계보다 오늘 처리할 일을 우선한다.

### 오늘 할 일

- 오늘 상담 예정
- 오늘 후속 확인
- 미처리 지원 요청
- 특이사항 보고 필요

### 빠른 실행

- 근로자 검색
- 상담·관찰 기록 추가
- 특이사항 보고

### 보조 정보

- 최근 작성 기록
- 자동 일일집계

민감한 상담 문장은 메인 화면에 길게 노출하지 않고, 이름·일정·상태 중심으로 보여준다.

## 10. 기록 수정과 보존

- 작성 직후 오타와 표현은 수정할 수 있다.
- 수정 시 이전 내용과 수정자를 기록한다.
- 잘못 작성한 기록은 삭제하지 않고 `정정됨`으로 표시한다.
- 담당자가 변경되거나 퇴사해도 기록은 회사에 남는다.
- 근로자가 퇴사한 후에도 과거 상담·지원 이력은 보존한다.
- 민감한 내용은 필요한 권한을 가진 사용자만 열람한다.

## 11. 처리 상태와 종료 기준

초기 처리 상태는 다음 4개로 단순화한다.

- 확인 필요
- 진행 중
- 후속 확인
- 완료

지원 건은 조치만 했다고 종료하지 않고 실제 효과를 한 번 확인한 뒤 완료한다.

다음 중 하나에 해당하면 완료할 수 있다.

- 요청한 지원조치가 실행되고 효과가 확인됨
- 근로자 재확인 결과 문제가 해소됨
- 일정 기간 관찰 후 추가 조치가 필요하지 않음
- 인사 기능이나 다른 부서 업무로 공식 이관됨
- 운영총괄 판단으로 종료됨

완료 시 `최종 조치`와 `확인 결과`만 간단히 남긴다.

## 12. 검색과 알림

### 검색

이름 검색을 중심으로 설계한다.

초기 필터:

- 근로자 이름
- 기록 유형
- 처리 상태
- 작성일 또는 기간
- 담당자
- 특이사항 여부

고급 검색과 복잡한 통계는 실제 기록량이 많아진 뒤 추가한다.

### 알림

초기 알림은 다음 3개에만 적용한다.

- 오늘이 후속 확인일인 건
- 일정 기간 미처리된 건
- 운영총괄 확인이 필요한 특이사항

일반 상담·관찰·완료 기록에는 별도 알림을 보내지 않는다. 문자·이메일보다 플랫폼 내부 표시를 우선한다.

## 13. 초기 제외 범위

다음은 초기 버전에 포함하지 않는다.

- AI 자연어 상담 요약
- 복잡한 부서 협의 워크플로우
- 다단계 승인선과 세분화된 상태값
- 복잡한 통계·분석 대시보드
- 외부 문자·이메일 자동 알림
- 의료적 진단이나 치료 판단 기능

전문 자격이 없는 수습담당자는 상담 보조, 관찰, 사실 기록, 지원 연결, 후속 확인을 수행하며 진단·치료·인사결정은 하지 않는다.

## 14. 개발 검수 기준

- 인사 기능과 근로자지원 권한이 분리되어 있는가
- 인사카드가 3개 탭 중심으로 단순하게 구성되는가
- 상담 원문이 다른 부서에 자동 공개되지 않는가
- 일반 상담기록이 운영총괄 대시보드에 노출되지 않는가
- 특이사항만 운영총괄 판단사항으로 표시되는가
- 근로자별 기록에서 일일 집계가 자동 생성되는가
- 후속 확인과 장기 미처리 건을 쉽게 찾을 수 있는가
- 기록 삭제보다 정정과 변경이력 보존을 적용하는가
- 실제 운영에서 필요성이 확인되지 않은 기능을 선행 개발하지 않는가
