# 운영총괄 홍보 대시보드 및 승인 업무 v1

상태: **확정**

이 문서는 태장 업무 플랫폼 홍보 모듈에서 운영총괄이 수행하는 승인·판단·방향 설정 업무와 대시보드 구성을 정의합니다.

## 1. 역할 원칙

운영총괄은 홍보팀의 모든 글을 다시 작성하거나 일반 콘텐츠를 전부 재검토하지 않습니다.

운영총괄은 다음 업무에 집중합니다.

- 중요한 콘텐츠의 사실관계와 회사 표현 판단
- 모회사·인증·고용·수치·금액 관련 검토
- 대표이사에게 올릴 안건의 선별과 사전 정리
- 홍보 기획안 승인
- 회사 일정과 홍보 방향의 연결
- 채널별 최소 발행량과 팀 운영 상태 확인
- 위험한 공개 자료 차단
- 승인 지연 사유와 재확인 날짜 기록
- 홍보팀장이 판단하기 어려운 문제 해결

운영총괄은 다음 업무를 직접 수행하지 않는 것을 원칙으로 합니다.

- 일반 콘텐츠 전부 재검토
- 직원에게 직접 세부 수정 지시
- 모든 글의 맞춤법 교정
- 사진 폴더 일일 정리
- 게시 시간·해시태그의 일상적 결정
- 최소 발행량을 개인 직원 평가에 사용

## 2. 하루 업무 흐름

```text
로그인
→ 오늘 결정해야 할 중요 업무 확인
→ 중요 콘텐츠 승인·보완 요청
→ 대표이사 확인 후보 정리·전달
→ 홍보 기획안 승인 또는 검토 보류
→ 장기 승인 대기 업무 확인
→ 채널별 최소 발행 현황 확인
→ 공개 홈페이지 주요 오류·노출 종료 확인
→ 홍보팀장 판단 요청 처리
→ 다음 주 홍보 방향 전달
→ 미처리 업무의 재확인일 또는 내일 우선 처리 설정
```

## 3. 첫 화면 핵심 현황

운영총괄 로그인 첫 화면에는 다음을 우선 표시합니다.

- 중요 콘텐츠 승인 건수
- 대표이사 확인 후보 건수
- 기획안 승인 건수
- 보완 후 재요청 건수
- 3일 이상 장기 대기 건수
- 긴급 확인 건수
- 오늘 주요 회사 일정

홍보 승인 업무는 회사 일정과 함께 보여 주어 판단에 필요한 맥락을 제공합니다.

## 4. 중요 콘텐츠 승인

중요 콘텐츠 화면은 본문 전체보다 먼저 다음 요약을 제공합니다.

- 작성자
- 홍보팀장 1차 검토 상태
- 중요 콘텐츠로 분류된 사유
- 홍보팀장 확인 의견
- 사실관계 확인 여부
- 인물 사진·수치·기관명·로고 등 주의 항목

운영총괄의 기본 행동은 다음과 같습니다.

- 승인 후 게시
- 예약 게시 승인
- 홍보팀장에게 보완 요청
- 대표이사 확인 요청
- 게시 보류
- 공개 중인 콘텐츠의 숨김 또는 수정 요청

## 5. 보완 요청 원칙

운영총괄은 홍보직원에게 직접 보완을 지시하지 않고 홍보팀장에게 전달합니다.

```text
운영총괄 보완 요청
→ 홍보팀장
→ 팀장이 직접 수정하거나 직원에게 보완 배정
→ 팀장 재검토
→ 필요한 승인 단계 재진행
```

보완 의견은 문제점과 수정 방향을 구체적으로 남깁니다.

## 6. 대표이사 확인 요청

대표이사에게는 초안을 그대로 넘기지 않습니다. 운영총괄이 먼저 다음을 정리합니다.

- 안건 요약
- 대표이사 확인이 필요한 이유
- 사실·기관명·인증번호·수치 확인 결과
- 운영총괄 검토 의견
- 대표이사가 최종 판단할 사항
- 원문 보기

대표이사 확인 대상 예시는 다음과 같습니다.

- 대표이사 명의 글
- 회사의 중대한 공식 입장
- 주요 모회사 관련 발표
- 중요한 협약·계약·경영 수치
- 사고·분쟁·언론 대응

## 7. 기획안 검토

유튜브 등 기획안 검토 시 다음을 확인합니다.

- 콘텐츠 목적과 핵심 메시지
- 태장의 고용·품질·사업 가치와의 연결
- 확보된 사진·영상 자료
- 추가 촬영 필요 여부
- 공개 위험 요소
- 게시 목표일

기획안이 바로 승인되기 어려우면 답변 없이 방치하지 않고 다음을 기록합니다.

- 검토 보류 사유
- 재확인 날짜
- 선행 조건

## 8. 채널 운영 현황

운영총괄은 홈페이지·네이버 블로그·인스타그램·X·유튜브의 최소 발행 현황을 팀 단위로 확인합니다.

발행량이 부족한 경우 직원에게 직접 지시하지 않고 홍보팀장에게 운영 방향을 전달합니다.

예:

- 기존 블로그에서 파생 콘텐츠 제작
- 행사 사진에서 짧은 현장 기록 제작
- 새로운 저품질 게시물의 억지 작성 금지
- 합리적 예외 사유 기록 허용

최소 발행량은 개인 평가 점수가 아니라 팀 운영 누락 방지 기준입니다.

## 9. 홈페이지 주요 상태 확인

운영총괄은 홈페이지의 모든 링크를 매일 직접 점검하지 않습니다. 홍보팀장이 등록한 중요 오류와 시스템 점검 결과만 확인합니다.

- 긴급 오류
- 주요 링크 오류
- 오래된 핵심 공지
- 외부 원문 연결 실패
- 민감 콘텐츠 공개 중단 요청

행사가 끝난 공지는 삭제보다 `메인 노출 종료·기록관 유지`를 우선합니다.

## 10. 홍보팀장 판단 요청

홍보팀장이 공개 여부를 판단하기 어려운 사진·문구·기관 정보 등을 `판단 요청`으로 보낼 수 있어야 합니다.

운영총괄은 다음 중 하나로 답변합니다.

- 사용 가능
- 수정 후 사용
- 상대 기관 확인 후 사용
- 사용 보류
- 사용 금지

판단 사유와 대체 방법을 함께 남깁니다.

## 11. 알림과 긴급도

모든 알림을 같은 수준으로 보내지 않습니다.

- 일반 승인 요청: 오늘 안에 확인 권장
- 예약 게시 마감 임박: 지정 시각 전 확인 필요
- 긴급 공식 대응: 즉시 확인 필요

퇴근 직전 들어온 일반 요청은 `내일 우선 확인`으로 넘길 수 있습니다. 시스템은 불필요한 야근을 유도하지 않습니다.

## 12. 주간 홍보 방향

운영총괄은 게시물 문장을 일일이 정하지 않고 회사가 이번 주 무엇을 보여 줄지 방향을 정합니다.

- 중심 홍보 주제
- 필수 채널
- 선택 채널
- 회사 일정과 연결할 메시지
- 유튜브 활용 검토 여부

홍보팀장은 이 방향을 글감 가이드와 팀 업무로 구체화합니다.

## 13. 대시보드 배치 순서

1. 오늘 결정해야 할 업무
2. 긴급·마감 임박 승인
3. 중요 콘텐츠 승인
4. 대표이사 확인 후보와 진행 상태
5. 기획안 승인·검토 보류
6. 3일 이상 장기 대기 업무
7. 채널별 최소 발행 현황
8. 이번 주 주요 행사와 홍보 방향
9. 홈페이지 주요 오류·노출 종료 요청
10. 홍보팀장 판단 요청
11. 오늘 처리 결과와 내일 이어보기

## 14. 상태와 기록

운영총괄 업무에는 다음 상태가 필요합니다.

- 승인 대기
- 보완 필요
- 대표이사 확인 대기
- 검토 보류
- 재확인 예정
- 승인 완료
- 게시 보류
- 내일 우선 확인

모든 승인·보완·보류·수정·대표이사 전달 이력은 기록합니다.

## 15. 개발 검수 기준

- 중요 안건만 운영총괄에게 올라오는가
- 운영총괄이 직원에게 직접 세부 지시하지 않는 흐름인가
- 대표이사에게 요약과 운영총괄 검토 의견이 함께 전달되는가
- 검토 보류 사유와 재확인 날짜를 기록할 수 있는가
- 채널 최소 발행량이 개인 평가가 아닌 팀 운영 현황으로 표시되는가
- 홈페이지 중요 오류와 노출 종료를 처리할 수 있는가
- 퇴근 직전 일반 요청을 다음 날로 넘길 수 있는가
- 역할에 맞는 업무 가이드를 현재 화면에서 즉시 열 수 있는가

## 16. 확정 문장

> 운영총괄은 홍보팀의 일반 콘텐츠를 다시 관리하는 사람이 아니라, 중요한 대외 표현과 회사 정보의 위험을 판단하고, 대표이사 확인 안건을 사전 정리하며, 회사 일정과 홍보 방향을 연결하는 최종 실무 책임자다. 운영총괄 대시보드는 오늘 결정할 중요 안건, 장기 대기, 채널 운영 현황과 판단 요청을 중심으로 구성한다.
