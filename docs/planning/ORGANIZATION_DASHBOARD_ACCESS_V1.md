# 태장 현재 조직·대시보드·접근 방식 v1

상태: **확정**

이 문서는 태장의 현재 실제 인원 배치, 조직별 책임자, 공석·겸임 상태, 사용 기기와 로그인 방식을 대시보드 설계 기준으로 고정합니다.

## 1. 기본 원칙

- 조직, 직책, 시스템 역할, 배치 상태를 서로 분리해 관리합니다.
- 부서는 인원이 없어도 조직도에 유지할 수 있습니다.
- 인원이 없는 부서는 `공석` 또는 `운영 예정`으로 표시하며 별도 계정과 대시보드는 활성화하지 않습니다.
- 한 명만 있는 부서는 실제 직책 하나만 부여하고, 팀장과 일반직원 역할을 중복 등록하지 않습니다.
- 본인이 등록하거나 요청한 업무를 본인이 최종 승인하지 않습니다.
- 겸임자는 별도 계정을 만들지 않고 기존 개인 계정에 필요한 역할과 메뉴만 추가합니다.
- 모든 재직 근로자는 개인 모바일 로그인을 기본으로 합니다.
- 팀장·관리자·사무직은 필요에 따라 PC 관리화면을 함께 사용합니다.
- 공용 PC는 실제 업무 필요가 확인될 때 추가 배치하며, 공용 계정은 사용하지 않습니다.

## 2. 현재 조직과 인원 상태

### 대표이사

- 전사 최종 의사결정자
- 주요 인사, 계약, 단가, 중요 지출과 대외 발표 최종 승인
- 세부 실무보다 최종 승인·핵심 경영정보·중대한 예외사항 중심의 대시보드 사용

### 운영총괄

- 전사 운영 총괄
- 부서 간 조정, 일반 인사변경 승인, 공석 조직 임시 관리
- 대표이사에게 올릴 중요 안건 선별

### 운영총괄실

현재 상태:

- 인사·운영지원 담당 공석
- 운영총괄이 임시 수행

향후 담당자 역할:

- 인사 실무
- 입사·휴직·복직·퇴사 절차 준비
- 직원명부·서류·변경이력 관리
- 계정 발급·정지 요청과 완료 확인
- 운영총괄 승인 대기와 미완료 업무 정리

담당자는 준비·정리·확인·공식 반영을 수행하며 주요 판단과 최종 승인은 하지 않습니다.

### 근로자지원팀

현재 상태:

- 근로자지원 담당 1명
- 수습 상태
- 팀장 공석
- 주요 검토와 승인은 운영총괄이 수행

수습 담당자 권한:

- 상담·관찰·지원 기록 작성
- 현장 확인 일정 관리
- 지원 필요사항 등록
- 부서 협의와 운영총괄 검토 요청
- 후속 확인 기록

제한:

- 직무·근무시간 직접 변경
- 휴직·퇴사 결정
- 민감기록 전체 내보내기
- 다른 부서의 불필요한 인사정보 열람
- 지원 제안 최종 승인

### 생산팀

현재 상태:

- 정식 팀장 공석
- 오전반 장애인 근로자 5명
- 오후반 장애인 근로자 5명
- 생산팀 전용 PC 없음
- 다른 부서 팀장급이 현재 민화 작업을 임시 지시
- 민화작가가 주 2회 방문해 생산·작품 지도를 수행
- 민화작가에게 1층 민화작업실 내 별도 책상 제공

운영 방식:

- 생산 근로자는 개인 모바일 또는 공용 태블릿에서 오늘 업무, 작업방법, 공지와 일정을 읽고 확인합니다. 일반 근로자에게 지원요청·시작·완료·실적 입력을 요구하지 않습니다.
- 작업배정·작업수량·진행·완료 확인과 필요한 기록은 임시 업무관리자 또는 향후 생산팀장이 현장에서 처리합니다.
- 다른 부서 팀장에게는 `생산팀 임시 업무관리자` 시스템 역할을 한시적으로 부여할 수 있습니다.
- 임시 업무관리자는 생산 업무배정과 진행기록은 할 수 있지만 인사변경·근무시간 변경·휴직·퇴사 결정은 할 수 없습니다.
- 정식 생산팀장이 배치되면 임시 권한을 회수합니다.

민화작가:

- 직원이나 생산팀장으로 등록하지 않고 외부 지도자로 구분합니다.
- 초기에는 별도 로그인 없이 담당자가 지도내용을 대신 기록할 수 있습니다.
- 직접 로그인이 필요해지면 민화 일정·작품 진행·지도내용만 볼 수 있는 제한 계정을 검토합니다.
- 직원 인사정보, 장애정보, 상담기록, 급여, 근로계약과 회사 대외비에는 접근할 수 없습니다.

### 물류팀

현재 상태:

- 물류팀장 1명
- 팀장 전용 PC 1대 보유
- 오전반 장애인 근로자 2명
- 오후반 장애인 근로자 2명
- 공용 PC는 현재 필수가 아니며 필요 시 추가 배치

물류팀장 PC 기능:

- 온라인 주문 전체 관리
- 오전·오후반 작업배정
- 재고·입출고·발주 요청
- 포장·검수·배송 진행상태
- 미완료 업무의 다음 반 인계
- 결원·지원 필요사항 확인

물류 근로자 모바일 기능:

- 본인 작업·작업방법·준비물 확인
- 공지·일정·근무시간 확인
- 작업방법이 기억나지 않을 때 짧게 다시 확인

문제·도움 필요·자재 부족 등은 현장 반장 또는 관리자에게 직접 알리고, 관리자가 필요한 기록을 남깁니다.

일반 근로자에게 온라인 주문 전체, 거래처 정보와 전체 재고정보를 공개하지 않습니다. 본인 작업 수행에 필요한 범위만 보여줍니다.

### 홍보·콘텐츠실

- 회사 핵심 전략조직으로 취급합니다.
- 홈페이지, 네이버 블로그, 인스타그램, 유튜브, X, 보도자료와 공식 기록을 통합 관리합니다.
- ESG·사회공헌·장애인 일자리 성과와 사진·영상·디자인 자산을 관리합니다.
- 행정상 운영총괄 산하로 두되 중요한 대외 메시지는 대표이사에게 직접 보고할 수 있습니다.

### 영업팀

현재 상태:

- 이선영 영업담당 이사 1인
- 대표이사 직속

권한:

- 거래처·영업기회·상담·미팅 관리
- 견적·제안 초안 작성
- 생산·물류 협조 요청
- 계약 승인 요청
- 수주·매출 진행상태 확인

제한:

- 본인이 올린 계약을 본인이 최종 승인하지 않음
- 최종 단가와 주요 계약조건은 대표이사 승인

### 농장지사

현재 상태:

- 인원 없음
- 지사장 공석
- 운영 예정 조직

조직도에는 유지하되 별도 계정과 지사 대시보드는 활성화하지 않습니다. 농장 관련 계획·자산·문서는 대표이사 또는 운영총괄 계정에서 임시 관리합니다.

## 3. 모바일 개인 로그인 원칙

태장의 모든 재직 근로자는 개인 모바일 계정을 기본으로 가집니다.

일반 근로자 첫 화면:

- 오늘의 업무
- 작업방법
- 내 근무시간·일정
- 중요공지
- 자주 보는 안내

사용성 원칙:

- 큰 글씨와 큰 버튼
- 쉬운 표현
- 메뉴 수 최소화
- 본인에게 필요한 기능만 표시
- 최초 로그인 후 안전한 로그인 상태 유지 검토
- PIN 또는 생체인증 등 간편 재인증 검토
- 휴대전화 분실·휴직·퇴사 시 세션 종료와 계정 차단

## 4. 역할별 대시보드 원칙

- 대표이사: 최종 승인, 핵심 경영정보, 중대한 예외사항
- 운영총괄: 오늘 처리할 일, 중요 승인, 지연·위험, 부서 간 조정
- 인사·운영지원 담당: 입퇴사·휴복직·서류·계정 처리와 미완료 업무
- 부서 팀장: 소속 인원, 작업배정, 진행상태, 협의 요청과 보고
- 근로자지원 담당: 상담·관찰·지원 일정과 후속 확인
- 일반 근로자: 읽기·확인 중심의 오늘 업무, 작업방법, 공지와 일정

직급이 높을수록 열람 범위는 넓어지지만 기본 대시보드는 더 압축합니다.

## 5. 승인과 계정 차단

- 일반 현장 지원은 팀장 간 자체 협의가 가능합니다.
- 운영총괄에게는 `참고 공유`와 `결정 요청`을 구분해 올립니다.
- 공식 부서·직무 변경은 운영총괄 승인 대상으로 합니다.
- 근무시간·근로계약·휴직·복직·퇴사 등 중요한 변경은 대표이사가 최종 승인합니다.
- 퇴사 처리는 인사상태 변경과 계정 접근 차단을 하나의 통합 절차로 관리합니다.
- 퇴사 시 계정 정지, 활성 세션 종료, 작성·수정·승인·업로드 차단, 업무 재배정, 과거 기록과 감사로그 보존을 확인합니다.
- 모든 필수 항목이 완료되기 전에는 퇴사 처리 완료로 표시하지 않습니다.

## 6. 기준 문서 우선순위

일반 근로자 화면은 `GENERAL_WORKER_INFORMATION_BOARD_V1.md`, 현장 책임자의 배정·최소 기록은 `FIELD_LEAD_OPERATION_RECORDS_V1.md`를 우선합니다. 이 문서의 과거 입력 중심 표현은 위 기준에 따라 해석·대체합니다.

## 7. 개발 검수 기준

- 현재 인원 0명·1명 부서를 정확히 표현하는가
- 공석 조직에 불필요한 계정과 대시보드를 만들지 않는가
- 물류팀장 PC와 근로자 개인 모바일 화면이 구분되는가
- 생산팀의 PC 부재와 오전·오후반 구조가 반영되는가
- 일반 근로자가 본인 업무에 필요한 정보만 볼 수 있는가
- 임시 업무관리자 권한이 정식 팀장 권한과 구분되는가
- 민화작가가 외부 지도자로 제한되는가
- 수습 근로자지원 담당자에게 과도한 권한이 부여되지 않는가
- 퇴사·휴직 시 모바일 세션과 계정 접근이 즉시 통제되는가
