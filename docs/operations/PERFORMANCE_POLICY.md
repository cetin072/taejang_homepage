# 성능 원칙

사이트 품질과 필요한 기능을 희생하지 않되, 사용하지 않는 코드·자산·라이브러리와 같은 낭비를 남기지 않는다.

## 이미지

- 대형 원본 이미지를 그대로 웹에서 사용하지 않습니다.
- 사진은 적절한 크기의 WebP 또는 JPG를 검토합니다.
- 로고·아이콘은 가능하면 SVG를 사용합니다.
- 투명 배경이 필요한 래스터 이미지는 PNG를 사용합니다.
- 표시 크기보다 과도하게 큰 이미지는 지양합니다.
- 아래쪽 콘텐츠 이미지는 lazy loading을 검토합니다.
- 히어로 핵심 이미지는 무조건 지연 로딩하지 않습니다.
- 이미지 품질이 눈에 띄게 저하되는 과도한 압축은 금지합니다.

## CSS와 JavaScript

- 같은 기능의 중복 구현을 금지합니다.
- 사용하지 않는 CSS·JS는 제거합니다.
- 페이지마다 동일 코드를 복사하기보다 공통 코드를 재사용합니다.
- 단순 기능을 위해 대형 라이브러리를 추가하지 않습니다.
- 라이브러리가 품질·안정성·개발효율에 명확한 이점이 있으면 사용할 수 있습니다.
- 애니메이션은 정보 전달이나 사용성 목적이 있을 때만 사용합니다.

## 폰트

- 불필요한 여러 웹폰트 사용을 지양합니다.
- 실제 사용하는 굵기만 로드합니다.
- 브랜드 전용서체 파일을 공개 저장소에 임의 등록하지 않습니다.
- 로고 글자는 이미지 또는 공식 벡터 자산을 사용합니다.

## 성능 판단

- 파일 용량 하나만 보고 품질을 낮추지 않습니다.
- 모바일 환경을 우선 확인합니다.
- 개선 전후의 체감 속도와 화면 품질을 함께 판단합니다.
- 관리자 기능과 공개 홈페이지의 JavaScript를 불필요하게 함께 로드하지 않습니다.
- 관리자 시스템 도입 시 공개 페이지와 관리 기능의 코드 경계를 유지합니다.
