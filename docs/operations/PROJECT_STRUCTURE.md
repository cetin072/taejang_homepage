# 프로젝트 폴더 구조

현재 정적 사이트 구조를 기준으로 각 경로의 역할을 정리합니다. 현재 없는 폴더는 실제 자산 또는 문서가 필요할 때만 생성합니다.

```text
assets/
├── css/                 공개 홈페이지 공통 스타일
├── js/                  공개 홈페이지 공통 기능과 콘텐츠 표시 기능
└── images/
    ├── brand/           태장 공식 웹용 로고·브랜드 이미지
    ├── partners/        승인된 협력기업·모회사 웹용 로고
    ├── business/        사업 및 역량 소개용 이미지
    ├── activities/      활동 콘텐츠 이미지
    └── workplace/       일터 콘텐츠 이미지

docs/
├── brand/               공개 가능한 브랜드 기준과 실제 자산 목록
├── operations/          Codex·파일·성능·프로젝트 운영 기준
└── templates/           보고서·게시물·개발 요청 등 반복 템플릿
```

## 공통 원칙

- 동일 파일을 여러 폴더에 복사하지 않습니다.
- 파일의 실제 용도에 따라 한곳만 원본 위치로 지정합니다.
- 다른 위치에서 필요하면 해당 경로를 참조합니다.
- 임시 파일과 생성 중간 산출물은 커밋하지 않습니다.
- 내부 원본 자료의 보관 경로는 공개 저장소 밖에서 관리합니다.
- 향후 Supabase 관리자 기능을 추가할 때 공개 자산과 업로드 자산의 경계를 별도 설계합니다.
