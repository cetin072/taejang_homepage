# 자산 관리 기준

## 목적

홈페이지와 회사 자료에 쓰는 파일을 중복 없이 한곳에 보관하고, 공개 가능한 자료와 내부 보관 자료를 구분합니다.

`docs/operations/ASSET_POLICY.md`는 모든 자산에 적용하는 일반 원칙이며, `docs/brand/BRAND_ASSETS.md`는 실제로 등록된 브랜드 자산의 목록·상태·교체 필요 여부를 기록합니다.

## 공개 저장소에 보관 가능한 자료

- 홈페이지에서 실제 사용하는 이미지
- 개인정보를 제거한 공개용 문서
- 사용 승인이 확인된 로고
- 웹용 SVG, WebP, PNG, JPG
- Codex가 지속적으로 참고해야 하는 공개 기준 문서

## 공개 저장소에 보관하지 않는 자료

- 개인정보가 포함된 문서
- 계약서, 주주자료, 주민등록번호 등 민감자료
- 초상권 또는 사용 승인이 확인되지 않은 사진
- 대용량 AI, PSD, EPS 원본 전체
- 오래된 CI 패키지 전체
- 중복본, 저해상도본, 임시본
- 최종 여부가 확인되지 않은 시안

## 파일 판정 상태

- `approved`: 저장소 등록 및 홈페이지 사용 가능
- `internal-only`: 회사 내부 보관만 가능
- `pending`: 최신성·권리·사실 확인 필요
- `temporary`: 임시 사용 가능, 공식 원본 확보 필요
- `rejected`: 중복·품질 부족·불필요 자료

새 파일을 받으면 저장 전에 위 상태를 판정하고, `approved`가 아닌 자료는 공개 홈페이지에 적용하지 않습니다.

## 파일명 규칙

- 영문 소문자를 사용합니다.
- 단어 구분은 하이픈을 사용합니다.
- 공백, 괄호, 한글 파일명, `최종`, `진짜최종`, 숫자만으로 된 이름은 사용하지 않습니다.
- 파일명만 보고 회사·용도·형식을 이해할 수 있게 작성합니다.

예: `taejang-logo-horizontal.svg`, `bumhan-logo.svg`, `samhyun-logo.jpg`, `taejang-ci-guide-public.pdf`

## 이미지 관리

- 원본 표시 비율을 유지합니다.
- 로고를 임의로 늘이거나 찌그러뜨리지 않습니다.
- 웹 표시 크기에 맞는 해상도를 사용합니다.
- 투명 배경이 필요한 경우 SVG 또는 PNG를 사용합니다.
- 사진은 품질을 확인한 뒤 WebP 또는 적합한 JPG를 사용합니다.
- 같은 이미지의 불필요한 중복 저장을 금지합니다.

## 권리와 출처

각 브랜드·사진 자료에는 다음 정보를 실제 자산 목록 또는 내부 관리 목록에 기록합니다.

- 출처
- 제공자
- 제공일 또는 확인일
- 사용 승인 상태
- 홈페이지 적용 상태
- 교체 필요 여부
- 원본의 내부 보관 위치가 있다면 그 위치
