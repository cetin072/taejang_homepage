# 일반 근로자 작업방법 구현

상태: 구현 후보 — Draft PR 검토 전에는 운영 Supabase에 적용하지 않는다.

## 범위

일반 근로자가 작업방법을 읽고 확인하는 화면이다. 수정, 댓글, 완료, 진행률, 평가, 시험, 실적 입력은 제공하지 않는다.

## 데이터와 게시

- `work_guides`는 분류, 공개범위(전체/부서), 대표 이미지 URL·대체텍스트, 준비물, 완성 모습, 주의사항, 자주 하는 실수, 담당 안내, 상태·버전·변경사유를 보관한다.
- `work_guide_steps`는 1~7의 중복 없는 순서, 쉬운 문장, 선택 이미지 URL·대체텍스트, 주의사항, 상태를 보관한다.
- `procedure`는 게시할 때 게시 상태 단계가 3~7개여야 한다. 안전·회사생활 같은 `reference`는 1~7개를 허용한다.
- 게시 뒤 수정은 현재 행의 `version_no`를 증가시키며, 삭제보다 `inactive`를 사용한다.

## 권한과 Today 연결

- 일반 근로자는 게시 자료 중 전체 공개·본인 부서 공개·본인 Today 업무에 연결된 자료만 안전한 RPC로 읽는다.
- 초안, 사용 중지 자료, 작성자 UUID, 변경사유, 감사로그는 반환하지 않는다.
- Today 업무는 게시 상태 작업방법만 연결할 수 있다. 이후 사용 중지되면 Today에는 안전한 미연결 안내가 표시된다.
- 최고관리자·운영총괄은 전체, 부서 팀장은 본인 부서, 현장 책임자는 담당 작업반이 속한 부서 자료를 관리한다. 모든 쓰기는 guarded RPC가 재검증한다.

## 이미지와 접근성

이번 단계는 URL만 선택 입력한다. URL에는 대체텍스트가 필수이며, URL 오류는 글 안내로 대체한다. Storage 업로드·공개 쓰기 버킷·base64 저장은 구현하지 않았다. 후속 Storage PR은 `cover_image_url`과 단계 `image_url`을 보호된 파일 참조로 교체한다.

목록은 큰 분류 버튼을 사용한다. 상세는 모바일 한 화면 한 단계, 큰 이전·다음 버튼, 현재 단계 수, 키보드 포커스, 제목 구조와 이미지 대체텍스트를 제공한다.

## 감사와 검증

작업방법·단계 생성/수정/순서변경/게시/사용중지와 Today 연결 변경을 짧은 식별·상태 metadata로 기록한다. 본문, 이미지 URL 전체, 개인정보, 비밀값은 감사 metadata에 넣지 않는다.

GitHub Actions의 격리 Supabase에서 migration 재적용, DB lint, pgTAP, 실제 Auth/Data API/RLS와 기존 Today·Phase 1A 회귀를 실행한다. 운영 적용 전에는 별도 비운영 Supabase와 가상 계정으로 이미지 URL·권한·모바일 동작을 다시 확인한다.
