# 홈페이지 콘텐츠 공개 운영 절차

이 절차는 콘텐츠 작성과 코드 반영, 공개 승인의 책임을 분리하기 위한 기준입니다. 직원은 `main` 또는 Netlify 설정을 직접 수정하지 않으며, Preview 승인 전에는 병합하지 않습니다.

## 운영 순서

1. 직원이 [콘텐츠 작성 양식](CONTENT_SUBMISSION_TEMPLATE.md)을 작성합니다.
2. 사진이 있으면 [사진 운영 가이드](PHOTO_GUIDE.md) 기준으로 전달합니다.
3. 내부 검토자가 개인정보와 공개 승인 여부를 확인합니다.
4. 개발 담당자 또는 Codex가 `content.js`에 반영합니다.
5. `node scripts/validate-content.js`를 실행합니다.
6. `git diff --check`와 필요한 정적 검사를 실행합니다.
7. 작업 브랜치와 Draft PR을 생성하거나 업데이트합니다.
8. Deploy Preview가 생성되었는지 확인합니다.
9. PC·모바일 화면을 검토합니다.
10. 최종 승인자가 공개 여부를 승인합니다.
11. 승인 후에만 `main` 병합을 진행합니다.
12. 실제 홈페이지에서 반영 상태를 확인합니다.

## 역할 구분

### 콘텐츠 작성자

- 사실관계 확인
- 제목·본문·사진 설명 작성
- 인물·기업명·로고의 공개 승인 정보 전달

### 내부 검토자

- 개인정보·문서 노출 여부 확인
- 기업명·로고 공개 승인 확인
- 날짜·시제·공개 가능 여부 검토

### 반영 담당자

- ID·카테고리·데이터 구조 확인
- 사진 경로·alt·안내박스 처리
- 자동 검사·정적 검사·Preview 확인

### 최종 승인자

- 외부 공개 최종 판단
- Deploy Preview 승인
- `main` 병합 승인

## 금지 및 보류 사항

- 직원이 `main`에 직접 수정하거나 병합하지 않습니다.
- 직원이 Netlify 설정·운영 도메인·환경 변수를 변경하지 않습니다.
- Preview 승인 없이 병합하지 않습니다.
- 운영 도메인 확정 전에는 canonical, Open Graph URL, sitemap, robots, 검색엔진 등록을 변경하지 않습니다. 자세한 상태는 [SEO·운영 도메인 체크리스트](SEO_DOMAIN_CHECKLIST.md)를 확인합니다.

공개 전 최종 확인에는 반드시 [홈페이지 공개 전 최종 검수 체크리스트](PUBLISH_CHECKLIST.md)를 사용합니다.
