# 배포 운영 가이드

## 현재 방식

이 사이트는 순수 HTML/CSS/JavaScript 정적 사이트이며, Netlify가 저장소 루트를 그대로 배포합니다.

- 배포 설정: `netlify.toml`
- publish 경로: `.`
- 별도 빌드 명령: 없음

## 변경과 배포 절차

1. 기능 브랜치에서 변경합니다.
2. 로컬에서 페이지, 이미지, 링크, 모바일 화면을 확인합니다.
3. 변경 목적과 확인 결과를 정리합니다.
4. PR을 생성해 검토를 받습니다.
5. 사용자 승인 후 `main`에 반영합니다.
6. 사용자 승인 후 Netlify 운영 배포 상태와 실제 URL을 확인합니다.

## 금지·주의 사항

- `main`에 직접 커밋하지 않습니다.
- 사용자 승인 없이 Netlify 운영 배포를 실행하지 않습니다.
- 도메인, 리디렉션, 환경 변수, 접근 권한은 개발자만 변경합니다.
- API 키·토큰·비밀번호는 저장소에 기록하지 않습니다.

## 배포 전 확인 목록

- 핵심 페이지가 열리는가
- 메뉴·버튼·전화·이메일·지도 링크가 동작하는가
- 사진과 공유 이미지가 누락되지 않았는가
- 모바일 화면에서 레이아웃이 깨지지 않는가
- 날짜, 연락처, 주소, 인증 표기가 최신인가
- 개인정보·미승인 사진·비공개 정보가 포함되지 않았는가

## 장애 대응 원칙

운영 화면 이상이 발견되면 즉시 변경을 멈추고, 최근 변경 내용·발생 시점·영향 페이지를 기록한 뒤 개발자가 원인을 확인합니다. 긴급 복구나 이전 배포본 전환도 사용자 승인 범위에서 진행합니다.
