## 2. 개발자가 기술적으로 결정 가능한 항목

아래 항목은 사용자 결정의 범위·비용·운영 조건 안에서 개발자가 설계하고, 보안·비용 영향이 생기면 다시 사용자 승인을 받습니다.

| 항목 | 기술 결정 기준 | 승인 재요청 조건 |
| --- | --- | --- |
| 코드 구조 | 공개 사이트와 관리자 코드 경계, 테스트 가능성 | 대규모 프레임워크 전환 |
| 내부 파일 키 | UUID·충돌 방지·경로 안전 | 공개 파일명 정책 변경 |
| DB 인덱스 | 검색·정렬 성능 | 비용·보존 정책 영향 |
| 정적 JSON schema | 기존 ID·URL 호환, 확장성 | 공개 URL·콘텐츠 구조 변경 |
| 이미지 파생 크기 | 화면 품질·모바일 성능 | 원본 삭제·품질 저하 가능성 |
| 오류·모니터링 구현 | 비밀·개인정보 마스킹 | 유료 서비스 도입 |
| 자동 테스트 | RLS·상태 전환·발행·회귀 | 운영 데이터 사용 필요 |

## 3. Phase 0 종료 조건

- 사용자 결정 항목 중 백엔드, 게시 방식, 로그인, 승인 단계, 원본 보관, 역할 책임자가 확정됨
- MVP 콘텐츠 유형과 화면 범위가 승인됨
- 월 예상 비용·Netlify 배포 영향·Supabase 백업 범위가 검토됨
- 개인정보 처리와 동의서 보관 책임자가 정해짐
- 개발·Preview·운영 환경 분리 수준이 결정됨
