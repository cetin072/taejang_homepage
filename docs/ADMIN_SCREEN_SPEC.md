## 공통 화면 원칙

- 역할에 없는 버튼은 숨기기만 하지 않고 서버 권한에서도 거부합니다.
- 저장·승인·게시·삭제는 처리 중 상태와 성공·실패 결과를 명확히 보여 줍니다.
- 목록 필터와 작성 임시저장은 브라우저 뒤로가기로 유실되지 않게 합니다.
- 키보드 포커스, 오류 요약, 필드별 오류, 충분한 대비와 터치 영역을 제공합니다.
- 모바일은 긴 글의 최종 편집보다 확인·승인·긴급 대응을 우선합니다.
- Preview에는 환경·revision·만료 여부를 표시하고 검색엔진 색인을 막습니다.

## 계정 상태 변경 확인

퇴사 처리 확인에는 다음 문구를 표시합니다.

> 이 계정은 즉시 로그인과 콘텐츠 작성이 차단됩니다.
>
> 기존 작성물과 변경 이력은 보존됩니다.

계정 복구 확인에는 다음 문구를 표시합니다.

> 이 계정을 다시 활성화합니다.
>
> 역할과 권한을 확인한 후 진행하세요.

일시 정지·퇴사 처리·복구는 처리 사유와 재확인을 요구합니다. 재배정되지 않은 초안·검토 대기·예약 게시물이 있으면 상태 변경 전에 경고하고, 완료 후 상태 변경·세션 종료·재배정 결과를 함께 보여줍니다.
