# 관리자 모드 로드맵

## 목표

비개발자 홍보팀 2~4명이 코드 수정 없이 홈페이지 콘텐츠를 관리할 수 있는 관리자 모드를 단계적으로 도입합니다.

## 역할과 권한

| 역할 | 담당 업무 | 제한 사항 |
| --- | --- | --- |
| 작성자 | 콘텐츠 작성, 임시저장, 검토 요청, 보고서 초안 | 직접 게시·설정 변경 불가 |
| 검토자 | 사실관계 검토, 수정 요청, 게시 승인 | 디자인·코드·보안·배포 수정 불가 |
| 게시 담당 | 승인된 콘텐츠 게시, 공유, 게시 결과 기록 | 설정·권한 변경 불가 |
| 최고관리자/개발자 | 계정·권한·백업·구조·보안·배포 관리 | 운영 기준과 사용자 승인 준수 |

초기에는 인원에 따라 한 사람이 여러 역할을 겸할 수 있지만, 작성과 최종 게시의 확인 절차는 분리합니다.

## 관리 대상

- 공지와 회사 소식
- 활동·일터 게시물과 사진
- 보도자료와 채용 공고
- 배너와 대표 이미지
- 회사 정보, 연락처, 공유용 문구
- 게시 이력과 간단 보고서 초안

## 단계별 계획

### 1단계: 정적 사이트 운영 기준 정비

- 콘텐츠 작성·검토·게시 기준 확정
- 공식 정보와 대외 표현 기준 문서 확정
- 사진·로고 사용 승인 관리 방식 확정
- 기존 정적 사이트는 개발자 변경 방식으로 유지

### 2단계: 관리자 요구사항 설계

- 작성·임시저장·검토 요청·승인·게시 흐름 정의
- 역할별 권한과 콘텐츠 상태 정의
- 게시물·사진·배너·회사 정보 데이터 구조 설계
- 백업, 수정 이력, 삭제 복구 기준 설계

### 3단계: Supabase 기반 관리자 시스템 검토

- Supabase Authentication으로 계정과 역할 관리
- Supabase Database로 콘텐츠·상태·이력 관리
- Supabase Storage로 승인된 이미지 관리
- Row Level Security로 역할별 접근 제한
- 공개 페이지와 관리자 페이지의 분리 운영

### 4단계: 제한된 시범 운영

- 홍보팀 2~4명 계정으로 작성·검토·게시 절차 점검
- 실제 게시물 몇 건으로 권한·승인·공유·보고서 초안 흐름 확인
- 오류, 교육 필요 사항, 운영 부담을 반영해 개선

## 개발자 전용 영역

다음 영역은 관리자 모드에서 수정할 수 없도록 분리합니다.

- 디자인 시스템과 화면 구조
- HTML/CSS/JavaScript 및 관리자 애플리케이션 코드
- 데이터베이스 구조, 보안 정책, API 키
- Netlify·Supabase 배포 설정, 도메인, 환경 변수
- 계정 권한, 백업, 복구 정책

## 도입 전 결정할 사항

- 공식 정보와 대외 표현 기준의 책임자
- 게시 승인권자와 긴급 수정권자
- 사진·로고 사용 승인 보관 방식
- 관리자 계정 발급·회수·비밀번호 정책
- Supabase 비용·백업·보안 검토
