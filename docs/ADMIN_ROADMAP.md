# 관리자 시스템 로드맵

## 목표와 현재 위치

비개발자 홍보팀 2~4명이 코드·GitHub·Netlify 설정을 수정하지 않고 콘텐츠를 작성·검토·승인·게시하도록 단계적으로 도입합니다. 현재는 **Phase 0 요구사항·의사결정 단계**이며 관리자 코드·Supabase 프로젝트는 아직 만들지 않습니다.

## 역할과 권한

| 역할 | 담당 업무 | 제한 사항 |
| --- | --- | --- |
| 작성자 | 콘텐츠 작성, 임시저장, 검토 요청, 보고서 초안 | 직접 게시·설정 변경 불가 |
| 검토자 | 사실관계 검토, 수정 요청, 게시 승인 | 디자인·코드·보안·배포 수정 불가 |
| 게시 담당 | 승인된 콘텐츠 게시, 공유, 게시 결과 기록 | 설정·권한 변경 불가 |
| 최고관리자 | 계정 발급·역할·일시 정지·퇴사 처리·복구·재배정·휴지통·감사 로그 관리 | 코드·DB·배포 직접 수정 불가 |
| 개발자 | 코드·DB 구조·보안 정책·배포·장애 대응과 기술 복구 | 콘텐츠 승인·퇴사 여부 판단과 분리 |

초기에는 인원에 따라 한 사람이 여러 역할을 겸할 수 있지만, 작성과 최종 게시의 확인 절차는 분리합니다.

## 관리 대상

- 공지와 회사 소식
- 활동·일터 게시물과 사진
- 보도자료와 채용 공고
- 배너와 대표 이미지
- 회사 정보, 연락처, 공유용 문구
- 게시 이력과 간단 보고서 초안

## Phase 0: 요구사항과 기술 결정

- 역할·권한·콘텐츠 유형·상태 전환 확정
- 정적 JSON 발행 방식과 Supabase 도입 승인
- 비용·백업·환경 분리·개인정보·동의서 보관 결정
- 관리자 화면·데이터 모델·보안 계획 승인

완료 기준: [의사결정 문서](ADMIN_DECISION_LOG.md)의 핵심 항목이 확정되고 MVP 범위·책임자가 승인됩니다.

## Phase 1: MVP

- 로그인과 직원 계정 발급·역할 설정
- 계정 일시 정지·퇴사 처리·기존 세션 차단·계정 복구
- 초안·검토·예약 게시물 담당자 재배정과 계정 상태 감사 로그
- 공지·우리의 일터·태장의 활동 작성과 사진 업로드
- 임시저장·검토 요청·수정 요청·승인·게시
- 실제 홈페이지 Preview와 승인 revision 정적 발행
- 휴지통·복원·기본 revision·감사 로그

완료 기준: 작성자·검토자·게시 담당의 실제 시나리오, `suspended`·`departed` Auth/API/RLS/Storage 차단, 세션 회수, 재배정, 게시·복구, 모바일 확인, 직원 매뉴얼이 통과합니다.

## Phase 2: 콘텐츠·미디어 확장

- 보도자료·채용공고·회사소개서 PDF·자료실
- 배너·팝업·예약 게시
- 미디어 원본·웹본·썸네일·중복 탐지 강화
- 첨부파일 검사·버전·이전 공개본 관리
- 보존 의무와 별도 승인을 전제로 예외적 계정 영구삭제 절차 검토

## Phase 3: 회사정보·보고·운영 고도화

- 회사 기본정보·대표 인사말·FAQ 관리
- 게시 통계·월간 보고·백업 UI
- 계정·권한·감사 로그·복구 기능 고도화
- 정기 권한 검토와 운영 지표

## Phase 4: 자동화·기업 협력 운영

- 이메일 알림·고급 검색·월간 보고 자동화
- 기업 협력·문의 배정·처리·회신 기록
- 공개 콘텐츠와 내부 협력 정보 분리 보고
- 필요 시 외부 업무 시스템 연계 검토

## 기술 방향

권장 구조는 **Supabase 관리자 + 승인 콘텐츠 정적 JSON 발행**입니다. 공개 페이지는 마지막 승인 정적본을 유지해 관리자 장애 영향을 받지 않도록 하고, 게시를 묶어 Netlify 배포 횟수를 관리합니다.

## 개발자 전용 영역

다음 영역은 관리자 모드에서 수정할 수 없도록 분리합니다.

- 디자인 시스템과 화면 구조
- HTML/CSS/JavaScript 및 관리자 애플리케이션 코드
- 데이터베이스 구조, 보안 정책, API 키
- Netlify·Supabase 배포 설정, 도메인, 환경 변수
- 계정 권한, 백업, 복구 정책

## 도입 전 결정할 사항

- 공식 정보와 대외 표현 기준의 책임자
- 게시 승인권자와 긴급 수정권자
- 사진·로고 사용 승인 보관 방식
- 관리자 계정 발급·회수·비밀번호 정책
- 마지막 활성 최고관리자 보호와 계정 복구 책임자
- Supabase 비용·백업·보안 검토

상세 결정은 [관리자 시스템 요구사항](ADMIN_SYSTEM_REQUIREMENTS.md), [화면 명세](ADMIN_SCREEN_SPEC.md), [데이터 모델](ADMIN_DATA_MODEL.md), [보안 계획](ADMIN_SECURITY_PLAN.md), [의사결정 문서](ADMIN_DECISION_LOG.md)를 따릅니다.
