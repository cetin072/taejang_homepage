# 콘텐츠 작성·게시 가이드

## 적용 대상

홈페이지의 회사 소개, 공지, 활동 소식, 보도자료, 채용 공고, 배너, 사진과 공유 문구에 적용합니다.

## 역할 분리

향후 비개발자 홍보팀 2~4명은 다음 업무를 담당합니다.

- 콘텐츠 작성
- 임시저장
- 검토 요청
- 승인된 콘텐츠 게시
- 공유용 문구 작성·공유
- 활동 및 게시 결과 보고서 초안 생성

디자인 시스템, HTML/CSS/JavaScript 코드, 보안, 데이터 구조, 배포 설정은 개발자만 수정합니다.

## 작성 절차

1. 사실관계와 근거 자료를 확인합니다.
2. 제목, 게시일, 본문, 사진, 담당자를 입력하고 임시저장합니다.
3. 검토 요청 시 변경 목적과 확인이 필요한 내용을 함께 남깁니다.
4. 검토·승인 후 게시합니다.
5. 게시 URL, 공유 문구, 게시일을 기록해 보고서 초안에 활용합니다.

## 문구 기준

- 쉽고 정확한 한국어를 사용하며 과장·추정·단정 표현을 피합니다.
- 회사명, 대표자, 연락처, 주소, 인증·선정 표기, 날짜는 게시 전 재확인합니다.
- 장애인 근로자 관련 표현은 존중하는 언어를 사용하고 개인의 정보나 상태를 불필요하게 공개하지 않습니다.
- 태장 공식 정보와 대외 표현 기준은 별도 문서로 관리합니다. 그 문서에 없는 신규 표현, 인증·수상·협력사 표기는 검토 후 사용합니다.

## 사진·로고 기준

- 초상권 및 촬영·사용 동의가 확인된 사진만 게시합니다.
- 미성년자, 근로자, 방문객 사진은 동의 범위와 공개 목적을 확인합니다.
- 로고는 사용 승인 범위와 최신 원본 파일을 확인한 뒤 사용합니다.
- 원본 파일명, 촬영일, 사용 승인 근거는 별도 내부 관리 목록에 기록합니다.

## 공개 금지 정보

다음 정보는 공개 저장소와 공개 페이지에 올리지 않습니다.

- 주민등록번호, 계좌번호, 연락처 등 개인식별 정보
- 주주명부, 내부 의사결정 자료, 계약서 원문, 비공개 재무자료
- 초상권 사용 여부가 확인되지 않은 사진
- 비밀번호, API 키, Netlify·Supabase 접속 정보
